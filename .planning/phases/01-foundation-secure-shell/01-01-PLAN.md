## Objective

Ship the Walking Skeleton — the smallest end-to-end slice that proves SoloCampaign's architecture (secure Electron + better-sqlite3 + Drizzle + electron-trpc v10 + React Query + react-router-dom 7 HashRouter) is correctly wired. The user can launch the app, create a campaign through the modal, see it in the card grid, navigate to `/campaign/:id`, and find the campaign still present after a restart.

This plan also establishes the **router skeleton** with `secrets`/`prefs`/`window` sub-routers as empty `t.router({})` placeholders so downstream plans (01-03 / 01-04 / 01-05) can add procedures without touching `router.ts` (interface-first parallelization).

## Context

**User Story:** **As a** solo D&D player, **I want to** create my first campaign in a secure local app, **so that** I have a persistent home for my future character, world, and AI DM.

Locked decisions implemented here (per CONTEXT.md):
- **D-01:** Fork `daltonmenezes/electron-app` via `degit` (no git history inheritance)
- **D-02:** Strip all demo pages, components, and `electron-router-dom` routes before building anything
- **D-04:** Card grid 2–3 cols via CSS grid `repeat(auto-fill, minmax(280px, 1fr))`
- **D-05:** Create modal with name field + Create button only
- **D-06:** `react-router-dom@7` `HashRouter` (NOT `electron-router-dom` which ships in boilerplate)
- **D-07:** Empty state shows "Start your first campaign" CTA — do NOT auto-open modal on first launch

Key research pins (per RESEARCH.md):
- `@trpc/server@^10` + `@trpc/client@^10` (electron-trpc 0.7.1 does NOT support v11 — Pitfall #2)
- `react-resizable-panels@^3` (v4 breaks shadcn's Resizable wrapper — Pitfall #1, even though Resizable isn't used in this plan, the pin lives in package.json now to avoid 01-04 needing a downgrade)
- `superjson` transformer wired into tRPC create + ipcLink (so Date round-trips)
- senderFrame URL validation in `createIPCHandler({ createContext })` (RESEARCH §5)
- BrowserWindow flags MUST appear literally: `contextIsolation: true`, `sandbox: true`, `nodeIntegration: false`
- CSP header set via `session.defaultSession.webRequest.onHeadersReceived` (RESEARCH §5)
- Boilerplate ships `electron-router-dom` (NOT react-router-dom) — Pitfall #11 — first task replaces it
- Boilerplate ships Biome (NOT ESLint+Prettier) — keep Biome per Open Question #3

This plan does NOT include: Drizzle migrations (raw `CREATE TABLE IF NOT EXISTS` is used; 01-02 replaces it), single-instance lock (01-02), backup rotation (01-02), integrity_check (01-02), SecretStorageService (01-03), split-panel layout (01-04), 5-tab shell (01-04), custom title bar (01-05), window size persistence (01-05), real app icons (01-06), unit tests (01-07), packaged smoke builds (01-08).

<threat_model>

| Threat | STRIDE | Mitigation in this plan |
|--------|--------|------------------------|
| T-RCE-01: Renderer XSS escalates to Node RCE via nodeIntegration | E (Elevation of Privilege) | `webPreferences` literally sets `contextIsolation: true`, `sandbox: true`, `nodeIntegration: false`, `nodeIntegrationInWorker: false`, `nodeIntegrationInSubFrames: false`, `webSecurity: true`, `allowRunningInsecureContent: false`, `experimentalFeatures: false` |
| T-RCE-02: Malicious resource loads via permissive CSP | I/E | CSP set on every response via `session.defaultSession.webRequest.onHeadersReceived` with `default-src 'self'`, `object-src 'none'`, `base-uri 'none'`, `form-action 'none'`; LLM hosts (api.openai.com, generativelanguage.googleapis.com, localhost:*, 127.0.0.1:*) pre-allowed in `connect-src` for Phase 3 |
| T-IPC-01: Untrusted renderer fakes IPC payloads to write arbitrary SQL | T (Tampering) | Every tRPC procedure has a Zod `.input()` schema; Drizzle uses parameterized queries (no raw SQL on user input); `createContext` validates `event.senderFrame.url` starts with `file://`, `app://`, or `http://localhost:*` |
| T-DB-01: Path traversal in campaign name → arbitrary file write | T | Zod schema enforces `z.string().trim().min(1).max(80)`; campaign id is UUID v4 (server-generated, never user-supplied); no filesystem path is derived from `name` in this plan |
| T-IPC-02: IPC frame from external URL (e.g. webview navigation) | E | senderFrame URL validation in `createContext` rejects any frame URL that isn't the local renderer; failure throws `Error('IPC sender frame URL not allowed')` |

</threat_model>
