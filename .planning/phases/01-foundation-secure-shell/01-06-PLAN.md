## Objective

Fix four code review findings that VERIFICATION.md flagged as BLOCKER or WARNING severity. These are surgical changes — no new features, no structural changes, just defect corrections.

- **CR-01 (BLOCKER):** DB init failure in `app.whenReady()` is silently swallowed. The app continues running with a broken database. Fix: propagate the error via `dialog.showErrorBox()` then `app.quit()`.
- **CR-03 (BLOCKER):** `SecretStorageService` calls `app.getPath('userData')` at class property initialization time (module load), before `app.whenReady()` fires. On some platforms this throws or returns the wrong path. Fix: replace the eager field with a lazy getter.
- **CR-04 (WARNING):** IPC senderFrame validation unconditionally allows `http://localhost:*` in production builds. Fix: wrap the localhost allowance in `process.env.NODE_ENV === 'development'`.
- **CR-05 (WARNING):** B64 fallback `decrypt()` uses `.toString()` (defaults to UTF-8) instead of `.toString('ascii')` for base64-encoded content. Fix: add the explicit `'ascii'` encoding argument.

Purpose: Closes Gap 3 (CR-03 crash risk — FOUND-04) and Gap 4 (CR-04 security — FOUND-01 / SC-4) from VERIFICATION.md. Eliminates the two BLOCKER findings before packaged builds are tested in 01-07.
