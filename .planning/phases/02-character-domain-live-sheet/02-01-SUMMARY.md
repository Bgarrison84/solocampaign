## Deviations from Plan

### Known Issues

**1. [ABI Mismatch] charactersRepo.test.ts fails in vitest (not a code bug)**
- **Found during:** Task 3 execution
- **Issue:** `better-sqlite3` was rebuilt against Electron 41 ABI 145 in Phase 1. System Node.js 24.15.0 uses ABI 137. `new Database(':memory:')` throws `NODE_MODULE_VERSION 145 vs 137`. The test file is syntactically valid and collects 14 tests — the failure is purely environmental.
- **Why acceptable:** Plan acceptance criteria explicitly states "tests may pass or partially pass — passing is preferred, but file MUST be syntactically valid and runnable." Plan 02 will run these tests through the tRPC router layer which doesn't require direct better-sqlite3 bindings in vitest.
- **Mitigation path:** Plan 02 can add a vitest setup that rebuilds better-sqlite3 for system Node.js ABI, or tests can be converted to use the tRPC caller pattern (which avoids the direct Database() constructor call).

**2. [Rule 1 - Bug] Fixed `const * as schema` syntax error in charactersRepo.test.ts**
- **Found during:** Task 3, first test run
- **Issue:** Invalid JavaScript syntax `const * as schema = await import('./schema')` — `*` namespace import is not valid in a destructuring assignment.
- **Fix:** Moved `import * as schema from './schema'` to the top of the file as a static import.
- **Files modified:** src/main/db/charactersRepo.test.ts

**3. [Rule 1 - Bug] Fixed `void getLoader` reference at module level in contentLoader.test.ts**
- **Found during:** Task 2, first test run
- **Issue:** `getLoader` was a local function inside the `describe` block but referenced at module level, causing `ReferenceError`.
- **Fix:** Removed the erroneous `void getLoader` line.
- **Files modified:** src/main/db/contentLoader.test.ts

**4. [Rule 1 - Bug] Fixed node:fs mock shape for CJS default import**
- **Found during:** Task 2, second test run
- **Issue:** `vi.doMock('node:fs', () => ({ readFileSync: ... }))` fails with "No default export defined" because `contentLoader.ts` uses `import fs from 'node:fs'` (default import). The mock must return `{ default: { readFileSync } }`.
- **Fix:** Changed all fs mocks in contentLoader.test.ts to `{ default: { readFileSync } }` shape.
- **Files modified:** src/main/db/contentLoader.test.ts
