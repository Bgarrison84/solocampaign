  <action>
    Use the Read tool on the 2024 PHB PDF to extract content. Read the Species chapter (typically pages 30-50) to enumerate every species in the 2024 PHB. For each: build a Race object with id (kebab-case), name (display), size, speed, abilityScoreIncreases (per the 2024 ASI rules — species don't grant fixed ASIs in 2024, but record the species' fixed bonuses where applicable, otherwise empty array), darkvision (number in feet if granted, otherwise omit), traits (array of {name, description} with FULL description text — per D-06/D-30 the wizard shows the full stat block), languages (auto-granted strings), freeLanguageChoices (number of additional choices), source ("2024 PHB p.XX"). Where the species has lineage variants (e.g., Elf has Drow/High/Wood), model each lineage as its own Race entry with subrace + parentRace fields, and the parent record's availableSubraces lists child ids.

    Write resources/races.json as a single JSON array. Validate that it parses (use Node `JSON.parse(fs.readFileSync('resources/races.json'))` via Bash node -e).

    For resources/equipment.json: read the PHB Equipment chapter (weapons, armor, gear tables). Write a JSON object mapping `equipmentClassMap: { [className]: EquipmentPackageOption[] }`. Per D-12, each class gets 2-3 starting equipment packages (use the PHB class entries' starting equipment options — e.g., Fighter: chain mail+martial weapon+shield OR leather+longbow+20 arrows). Each package has id (kebab-case like "fighter-a"), label (human-readable), items[] (each {name, quantity, weight (lb as decimal), isMagic?: false for starting gear}), startingGold (gp awarded in addition to items, often 0 for class starting packages — some classes get gp instead). NOTE: equipment package data was already partially captured in classes.json startingEquipmentPackages. Decision: equipment.json is the authoritative store; classes.json may either repeat or reference by id. Per RESEARCH.md §Pattern 3 the content router exposes `content.equipment.listForClass({ className })` reading `loadContent().equipment[className]` — so this file must be shaped `{ [className]: EquipmentPackageOption[] }`. classes.json's startingEquipmentPackages may include just package IDs OR full duplication — duplicate the full data for simplicity (single source of truth per file is preferred; cross-references add complexity).

    For resources/spells-by-class.json: read the PHB Spellcasting chapter and each spellcaster's class entry. Write a JSON object matching `SpellSlotsByClass` type: `{ [className]: { [level: number as string]: { [slotLevel: string]: maxSlots } } }`. Cover levels 1-20 for: bard, cleric, druid, paladin, ranger, sorcerer, warlock (special — Pact Magic slot table), wizard. Per RESEARCH.md A6, authoring through level 20 saves Phase 5 rework. Half-casters (paladin, ranger) start spell slots at level 2. Warlock uses Pact Magic table from PHB. Non-spellcasters (fighter, barbarian, monk, rogue) are NOT keys in this file — the buildSpellSlots function returns {} for absent keys per Plan 01 behavior.

    Validate all three files parse as valid JSON.
  </action>

  </verify>
