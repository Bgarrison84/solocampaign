# Phase 03 Plan 01: Schema Foundation + Wave 0 Stubs Summary

**One-liner:** Drizzle migration 0002 adds 8 AI config columns to campaigns + messages table (cascade delete), with messagesRepo, campaignsRepo.updateAiConfig, and 7 Wave 0 test scaffolds.

## What Was Built

### Schema Extension (schema.ts)
- Added 8 AI config columns to `campaigns` table per D-07: `provider_type`, `endpoint_url`, `model_name`, `reference_docs` (JSON array, default `'[]'`), `dm_personality`, `strictness` (default `'balanced'`), `fallback_endpoint_url`, `fallback_model_name`
- No `api_key` column — security requirement D-08/D-23 (keys live in SecretStorageService only)
- Added `messages` sqliteTable: `id`, `campaign_id` (FK → campaigns ON DELETE CASCADE), `role`, `content`, `created_at`
- Exported `Message` and `NewMessage` inferred types

### Migration (0002_handy_spectrum.sql)
- Generated by `npm run db:generate` (drizzle-kit) — not hand-written
- Contains `CREATE TABLE \`messages\`` with `ON DELETE cascade` foreign key
- Contains 8 `ALTER TABLE \`campaigns\` ADD COLUMN` statements
- Zero `api_key` references
- Journal updated with idx 2 entry

### messagesRepo.ts
- `insert({ campaignId, role, content })` → returns full Message row
- `getLastN(campaignId, n)` → last N messages in chronological order; uses `rowid` DESC + reverse to handle same-millisecond timestamp ties
- `getByCampaignId(campaignId)` → all messages in insertion order

### campaignsRepo.ts
- Added `AiConfigInput` interface
- Added `updateAiConfig(campaignId, config)` method — sets all 8 AI config columns; serializes `referenceDocs[]` as JSON string

### Test Coverage
- `campaignsRepo.test.ts` — 9 tests green: CRUD operations + 4 AI config round-trip tests (SESS-05)
- `messagesRepo.test.ts` — 8 tests green: insert, getLastN ordering + limit + scope, cascade delete, getByCampaignId
- 5 Wave 0 stub files (28 `it.skip` tests, no hard errors): `llmProvider.test.ts`, `contextBuilder.test.ts`, `retryHandler.test.ts`, `referenceDocLoader.test.ts`, `ai.test.ts`

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 3 - Blocking] Rebuilt better-sqlite3 native module**
- **Found during:** Task 2 — first test run
- **Issue:** `better-sqlite3` native binary compiled against NODE_MODULE_VERSION 145 but test runner uses Node.js v24.15.0 (NMV 137)
- **Fix:** Ran `npm rebuild better-sqlite3` in the main project directory (worktree shares node_modules)
- **Impact:** Pre-existing; all existing DB tests had the same failure before this run

**2. [Rule 3 - Blocking] Installed @radix-ui peer dependencies**
- **Found during:** Task 2 — typecheck after implementation
- **Issue:** shadcn components added in Task 1 (checkbox, collapsible, tooltip) reference `@radix-ui/react-checkbox`, `@radix-ui/react-collapsible`, `@radix-ui/react-tooltip` which were not installed
- **Fix:** `npm install @radix-ui/react-checkbox @radix-ui/react-collapsible @radix-ui/react-tooltip --legacy-peer-deps`
- **Files modified:** `package.json`, `package-lock.json`

**3. [Rule 1 - Bug] rowid-based ordering in messagesRepo**
- **Found during:** Task 2 — first test run after rebuild
- **Issue:** `getLastN` used `desc(messages.createdAt)` but in-memory SQLite inserts in the same millisecond, so all rows get the same timestamp; DESC sort order was not stable
- **Fix:** Changed `getLastN` and `getByCampaignId` to use `rowid` for deterministic insertion order
- **Commit:** a6b293a (same task commit)
