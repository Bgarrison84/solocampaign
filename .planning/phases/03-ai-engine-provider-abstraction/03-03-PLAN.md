<objective>
Build the bridge between renderer and AI backend: the custom `ai:send-message` IPC streaming channel (the secure path where keys are decrypted in-process and only tokens cross to the renderer per D-23), the `window.aiStream` contextBridge surface, the `ai` tRPC router (listReferenceDocs + getMessages), the `campaigns.updateAiConfig` mutation (provider config + key storage), `prefs.isEncryptionAvailable`, and the in-memory per-session fallback state (D-19).

Purpose: tRPC v10 cannot stream over contextBridge, so streaming uses a dedicated IPC channel alongside tRPC (AI-SPEC §3). This plan wires the renderer's eventual `useAiStream` hook to the Wave 1 engine.
Output: Streaming IPC handler + preload surface + ai router + updated campaigns/prefs routers + session state module.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| renderer → ipcMain (ai:send-message) | untrusted payload + sender frame crosses here |
| main process → SecretStorageService | plaintext key materializes only here, per-request, never returned |
| main process → renderer (ai:token/finish/error) | only tokens + generic error strings cross back |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-03-01 | Information Disclosure (key leak) | ai:send-message handler | mitigate | Key decrypted in main only (D-23); never sent on ai:token/finish/error; never logged (only length/latency metrics). Acceptance asserts no key sent to renderer. |
| T-03-03-02 | Spoofing (IPC impersonation) | ai:send-message handler | mitigate | senderFrame.url validated against file://, app://, dev localhost allow-list — same as createIPCHandler context (T2) |
| T-03-03-03 | Tampering (streaming injection) | ai:send-message payload | mitigate | sendMessageSchema.parse validates campaignId (uuid) + content (max 10000) before any DB/LLM op (T3) |
| T-03-03-04 | Information Disclosure | ai:error payload | mitigate | Error surfaced to renderer is a generic message string; no stack trace, provider response body, or key |
</threat_model>
