## Spacing Scale

Inherited from Phase 1, unchanged. Multiples of 4 only.

| Token | Value | Usage in Phase 3 |
|-------|-------|-----------------|
| xs | 4px | Icon-text gaps, checkbox label gap, pip gaps |
| sm | 8px | Input field row gaps, wizard step padding, button group spacing |
| md | 16px | Story scroll horizontal padding, modal body padding, section gaps |
| lg | 24px | Wizard dialog body padding, chat panel vertical padding, large section breaks |
| xl | 32px | Outer layout padding (not added in Phase 3) |
| 2xl | 48px | (not used in Phase 3) |
| 3xl | 64px | (reserved, not used) |

**Phase 3 specific exceptions:**

- Story scroll prose area: `px-6 py-6` (24px horizontal + 24px vertical) — generous breathing room for novel-reading feel.
- Story scroll max-width: `max-w-[680px] mx-auto` — constrains prose line length to ~80 characters at 14px body size; wider than a book column but narrower than full panel width.
- Chat input textarea min-height: `min-h-[56px]` (56px = 4 text rows at 14px line-height 1.5, approximately 3.5 lines, grows automatically).
- Chat input area: `p-3` (12px) interior padding — not a standard token but justified for the input row (12 = 3×4, still on the 4-point scale).
- Player separator rule: `my-6` (24px vertical margin above and below the player entry block) — creates clear white space to mark the player's turn in the continuous scroll.
- Reference doc list: `max-h-[240px]` (240px = 60×4) — scroll threshold for 30-item list at 40px per item.
- Wizard dialog width: `max-w-[560px]` — wider than Phase 1's 480px to accommodate the two-column provider fields + reference doc list.
- Fallback collapsible trigger indent: `pl-0` on the trigger, `pl-4` on the revealed content — 16px indent matches existing `md` token.
- Gear icon button: `h-8 w-8` (32px) touch target — minimum accessible target size, consistent with Phase 1 title bar controls.

## Typography

Inherited from Phase 1 (4 roles, 2 weights). Phase 3 adds one usage pattern without new type roles.

| Role | Size | Weight | Line Height | Phase 3 Usage |
|------|------|--------|-------------|---------------|
| Body | 14px | 400 | 1.6 (22px) | AI narration prose, reference doc list items, wizard field helper text. **Note: story scroll uses 1.6 (not 1.5) — the extra leading improves long-form reading comfort** |
| Label | 14px | 600 | 1.4 (20px) | Wizard field labels, provider type picker options, "You:" separator prefix, wizard step labels, send button label |
| Heading | 20px | 600 | 1.3 (26px) | Wizard dialog title, wizard step heading, inline error block heading |
| Display | 28px | 600 | 1.2 (34px) | (not used in Phase 3 — declared here for completeness) |

**Additional type notes for Phase 3:**

- Player message text in story scroll: 14px / 400 / 1.5 — same as Body but in normal weight. The "You:" prefix is italic + muted gold color (see Color section).
- `"You:"` label: 14px / 400 / italic / `--muted-foreground` shifted to `--primary` (muted gold) — implemented as `text-sm italic text-primary/80`.
- Wizard step counter (e.g. "Step 2 of 3"): 14px / 400 / `--muted-foreground` — below the dialog title, not a separate heading.
- DM personality textarea placeholder and input: Body style (14px / 400).
- Reference document item title: 14px / 400 — no semibold; the checkbox itself provides enough visual affordance.
- Inline error block: Heading (20px / 600) for the error title, Body (14px / 400) for the descriptive message.
- Send button Ctrl+Enter hint: 12px / 400 / `--muted-foreground` — one allowed exception to the 4-role rule; this is microcopy below a button, matches Phase 2's precedent for sub-label helper text within interactive elements.

**Accent (muted gold) reserved EXCLUSIVELY for:**

1. Primary CTA button background — "Create Campaign" and "Save Changes" in wizard.
2. Active wizard step dot — filled gold circle (consistent with Phase 2 `WizardProgress` component).
3. Active tab trigger — bottom border + label color (unchanged from Phase 1).
4. "You:" separator prefix — `text-primary/80` (80% opacity gold) to subtly mark player entries without full gold brightness.
5. Selected reference document checkbox fill — `data-[state=checked]:bg-primary` (consistent with Phase 2 condition badge active state pattern).
6. Focus rings — 2px outline + 2px offset on all interactive elements.

**Accent is FORBIDDEN on:**
- AI narration text (white `--foreground` only — immersion requires no gold "system" color on the DM's words).
- Input field borders in default state.
- The gear icon in default state (`--muted-foreground` default, gold only on hover).
- Cancel / Back buttons.

**Color application for specific Phase 3 elements:**

- **Player separator rule:** `border-t border-border` (1px `--border` color) — neutral, not gold. The `You:` prefix already provides color identity.
- **Blinking cursor at stream tail:** `--foreground` color, 2px wide, `animate-pulse` — same color as surrounding text so it reads as a continuation.
- **Inline error block:** `bg-card` surface + `border-l-4 border-destructive` left accent + `border border-border` on other 3 sides — the destructive left stripe is the error signal without a red background.
- **safeStorage warning:** Amber `text-amber-500` + `bg-amber-950/40` — uses Tailwind v4 built-in amber scale; not `--destructive` (this is a warning, not an error).
- **Streaming timeout empty state (15s no token):** Same inline error block as provider failure (see D-18 / AI-SPEC guardrails).

#### S1a. `StoryScrollPanel` — Story Scroll

**Visual Treatment:**
- Background: `--background` (dark base, same as left panel surface)
- Overflow: `overflow-y-auto` with custom scrollbar: `scrollbar-thin scrollbar-thumb-border scrollbar-track-transparent` — Tailwind v4 scrollbar utilities; thin 4px scrollbar, visible only on hover/scroll
- Content wrapper: `max-w-[680px] mx-auto px-6 py-6` — centered prose column with generous breathing room
- No border between scroll area and input (the `border-t` is on `ChatInputArea` itself)

**AI Narration Block:**
- Each AI response is a block of `<p>` elements separated by `mb-4` (16px) paragraph breaks
- Text: `text-sm leading-[1.6] text-foreground` — 14px, 1.6 line-height for readability
- Rendered via `react-markdown` + `remark-gfm`: `**bold**` becomes `<strong>` with `font-semibold`, `*italic*` becomes `<em>` with `italic`, lists rendered as `<ul>/<ol>` with `pl-4 list-disc`
- Heading markdown (`##`, `###`) in AI responses: `text-base font-semibold text-foreground` — capped at Label size; do not let AI headings overshadow the UI hierarchy
- No timestamp visible per message (timestamps stored in DB; not surfaced in Phase 3)

**Player Entry Block:**
```
─────────────────────────────────────────────────────  (horizontal rule)
You:  [player message text here]
─────────────────────────────────────────────────────  (spacing below)
```
- Separator rule: `<hr className="border-t border-border my-6" />` — 1px `--border` color, 24px vertical margin above and below the full player entry block
- "You:" prefix: `<span className="text-sm italic text-primary/80 mr-2">You:</span>` — italic, 80% opacity muted gold
- Player message text: `<span className="text-sm leading-[1.6] text-foreground">` — same size as AI prose, not italic, not muted; visually continuous with AI narration but set apart by the rule and prefix
- The horizontal rule appears ABOVE the player entry; no rule appears between two consecutive AI paragraphs
- On mobile / narrow widths (not relevant for Electron desktop but noted): the rule and prefix scale gracefully because they're inline elements

**Empty State (no messages loaded yet):**
- Centered in the scroll area (full height centering via `flex items-center justify-center`):
  ```
  Begin your adventure.
  Type a message below to start the story.
  ```
  - Heading: `text-sm font-semibold text-muted-foreground` — intentionally small; doesn't compete with actual story text
  - Body: `text-sm text-muted-foreground/60` — further muted

**Streaming State — Blinking Cursor:**
- When tokens are arriving from `ai:token` IPC events, a `|` character (or a thin 2px div) is appended after the last rendered character
- Implementation: `<span className="animate-[blink_1s_ease-in-out_infinite] text-foreground">|</span>` where `blink` is a CSS keyframe:
  ```css
  @keyframes blink {
    0%, 100% { opacity: 1; }
    50% { opacity: 0; }
  }
  ```
  Declared in the Tailwind v4 `@theme` block or as a CSS class in the renderer's global CSS. Cursor is the same `--foreground` color as the surrounding text — it reads as the AI's writing position.
- The cursor is removed when `ai:finish` fires
- When streaming is paused (IPC token batch gap), the cursor remains visible; this is correct behavior — the player sees the AI is still composing

**Auto-Scroll Behavior:**
- The scroll area auto-scrolls to bottom when new tokens arrive IF the user was already at the bottom (within 100px of the bottom)
- If the user has scrolled up to re-read earlier content, auto-scroll is suspended — the user's scroll position is preserved
- Implementation: track `isUserScrolledUp` boolean via `onScroll` event; reset to `false` when user scrolls back to bottom; auto-scroll only fires when `!isUserScrolledUp`
- On `ai:finish`: always scroll to bottom regardless of `isUserScrolledUp` (the response is complete; player should see the end)
- On campaign load (message history rendered): scroll to bottom immediately (no animation — instant jump)
- After player submits a message: scroll to bottom immediately

#### S1b. `ChatInputArea` — Player Input

**File:** `src/renderer/src/components/ChatInputArea.tsx`

**Layout:**
```
┌──────────────────────────────────────────────┬──────────────┐
│  <Textarea>                                  │  [Send]      │
│  Auto-growing, min 56px, max ~112px          │  button      │
└──────────────────────────────────────────────┴──────────────┘
   Ctrl+Enter to send
```

- Container: `border-t border-border bg-background p-3 flex flex-row items-end gap-2`
- Textarea: `flex-1 min-h-[56px] max-h-[112px] resize-none bg-secondary border border-border rounded-md text-sm text-foreground placeholder:text-muted-foreground focus:ring-1 focus:ring-primary focus:border-primary transition-all overflow-y-auto`
  - `min-h-[56px]`: approximately 3 text lines at 14px / 1.5 line-height — enough for short messages
  - `max-h-[112px]`: approximately 6 lines — hard cap prevents the input from eating panel height
  - `resize-none`: no drag-resize handle; the textarea grows via JS `scrollHeight` adjustment
  - Growing behavior: on `onChange`, set height to `auto`, then to `scrollHeight`, capped at `max-h-[112px]`
  - Placeholder: `"What do you do?"`
- Send button: `Button variant="default" size="sm"` with `--primary` (gold) background
  - Label: `"Send"` — visible text, not icon-only
  - Width: fixed `w-16` (64px) — consistent with adjacent textarea
  - Height: `h-9` (36px) — aligned to the textarea's minimum height with `items-end` on the container
  - Disabled states:
    - Empty textarea: `disabled` — opacity 40%, cursor not-allowed
    - Streaming in progress (awaiting `ai:finish`): `disabled` — prevent double-send
    - Both: disabled with spinner icon replacing "Send" text while streaming
  - Loading state while streaming: `<Loader2 className="h-4 w-4 animate-spin mr-1" />` + `"Sending…"` — replaces "Send" label
- Ctrl+Enter hint: `<p className="text-[12px] text-muted-foreground mt-1">Ctrl+Enter to send</p>` — below the textarea, NOT inside it; appears immediately on first render (not only on focus)
- Plain Enter in textarea: inserts a newline (default `<textarea>` behavior — no override)
- `react-hotkeys-hook` binding: `useHotkeys('ctrl+enter', handleSend, { enableOnFormTags: ['TEXTAREA'] })` — fires when focus is anywhere in the component

**Keyboard Behavior:**
- `Ctrl+Enter` anywhere in the chat panel area: sends if textarea is not empty and not streaming
- `Enter` in textarea: newline (do NOT submit)
- `Tab` from textarea: moves to Send button
- `Enter` / `Space` on Send button when focused: sends

**Disabled State (no AI config):**
If a campaign has no AI provider configured yet (new campaign created before wizard, or config cleared), both textarea and Send button are disabled with an inline notice replacing the hint:
```
text: "Configure an AI provider to start playing."
```
`<p className="text-[12px] text-amber-500 mt-1">Configure an AI provider to start playing. <button className="underline">Open settings.</button></p>`
The "Open settings." button opens the gear icon modal (same as the gear icon in the action bar).

## Accessibility

| Surface | Requirement | Implementation |
|---------|-------------|----------------|
| Story scroll | Announce new AI content to screen readers | `<div aria-live="polite" aria-atomic="false">` wrapping the scroll content — progressive announcements as tokens arrive |
| Streaming cursor | Hidden from screen readers | `<span aria-hidden="true">|</span>` — sighted users see it; screen readers do not announce a blinking pipe |
| "You:" prefix | Readable by screen reader | No `aria-hidden`; reads as "You: [message text]" — accurate |
| Chat textarea | Label | `<Label htmlFor="chat-input">Your message</Label>` — visually hidden via `sr-only` (the textarea's purpose is visually obvious but label is required for AT) |
| Send button | Descriptive label | `aria-label="Send message"` when button label is icon-only; visible "Send" text means `aria-label` is optional but still recommended |
| Reference doc checkbox | Label | `<Label htmlFor={checkboxId}>{docTitle}</Label>` — clicking the label row toggles the checkbox |
| Collapsible trigger | Expanded state | `aria-expanded={isOpen}` on the trigger button |
| Inline error block | Alert role | `<div role="alert" aria-live="assertive">` — announces to screen readers immediately when error appears |
| Provider field group | Fieldset | `<fieldset>` + `<legend>` grouping related fields (OpenAI-compatible fields, Gemini fields, fallback fields) |
| Wizard step indicator | Step state | `aria-current="step"` on active dot; `aria-disabled="true"` on future dots |
| Gear modal | Focus trap | Radix Dialog handles focus trap automatically |
| Wizard modal | Focus trap | Radix Dialog handles focus trap |
| Gear icon button | Tooltip | Tooltip on hover/focus; `aria-describedby` linking to tooltip content |
| Large doc warning icon | Screen reader text | `<span className="sr-only">Large document — may exceed model context window</span>` in addition to the tooltip |
