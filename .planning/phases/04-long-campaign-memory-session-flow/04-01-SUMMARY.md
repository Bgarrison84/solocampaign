# Phase 4 Plan 1: Schema Foundation + sessionsRepo + Wave 0 Stubs Summary

**One-liner:** Drizzle sessions table (11 columns), migration 0003, sessionsRepo (11 methods), messagesRepo session-scoped queries, and 50 Wave 0 todo stubs across 4 test files for Phase 4's three-layer memory system.

## What Was Built

### Task 1: Schema Extension + scroll-area Component
- Extended `src/main/db/schema.ts` with:
  - `sessions` table (D-19): id, campaign_id FK cascade, session_number, started_at, ended_at (nullable), location, goal, context_notes, ai_recap, player_notes, is_summarized
  - `messages.sessionId` nullable FK to sessions (D-20) — declared after sessions table to resolve FK at compile time
  - `campaigns.rollingSummary` nullable text column (D-21)
  - Exported `Session` and `NewSession` types
- Installed `@radix-ui/react-scroll-area` and created `scroll-area.tsx` shadcn component
- Auto-fixed `contextBuilder.test.ts` Message fixtures to include `sessionId: null` (schema-driven TS error)

### Task 2: Migration 0003 (BLOCKING)
- Ran `npx drizzle-kit generate` producing `resources/migrations/0003_slippery_carnage.sql`
- Migration contains:
  - `CREATE TABLE sessions` with all 11 columns
  - `ALTER TABLE campaigns ADD rolling_summary text` (nullable)
  - `ALTER TABLE messages ADD session_id text REFERENCES sessions(id)` (nullable, no ON DELETE SET NULL)
- `_journal.json` updated with idx 3 entry

### Task 3: sessionsRepo + messagesRepo Extension + Wave 0 Stubs
- Created `src/main/db/sessionsRepo.ts` with 11 methods:
  - `create` (monotonic session_number via MAX query)
  - `getById`, `getActiveByCampaignId`, `getLastNCompleted`, `getOlderThan`, `list`
  - `end`, `saveRecap`, `updatePlayerNotes`, `markSummarized`
  - `getLastLocation` (pre-fill on next session start), `endAllActive` (app quit handler)
- Extended `messagesRepo.ts`:
  - Added `sessionId?: string | null` to `InsertMessageInput`
  - Included `sessionId` in `.values({...})` on insert
  - Added `getBySessionId(sessionId)` — all messages for recap generation
  - Added `getLastNForSession(sessionId, n)` — L1 overflow fallback
- Created 4 Wave 0 test stub files (50 todo stubs total):
  - `sessionsRepo.test.ts` — 29 stubs
  - `recapGenerator.test.ts` — 8 stubs
  - `sessions.test.ts` (router) — 13 stubs
  - `contextBuilder.test.ts` v2 describe block — 7 stubs (appended, existing v1 tests untouched)

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 3 - Blocking] shadcn CLI npm peer dependency conflict on scroll-area install**
- **Found during:** Task 1
- **Issue:** `npx shadcn@latest add scroll-area` failed because the shadcn CLI runs `npm install @radix-ui/react-scroll-area` without `--legacy-peer-deps`, and the known @trpc/react-query v10 / @tanstack/react-query v5 peer conflict blocked the install
- **Fix:** Installed `@radix-ui/react-scroll-area` directly with `npm install --legacy-peer-deps`, then manually created `scroll-area.tsx` following the existing shadcn component pattern (same structure as `checkbox.tsx`, `tooltip.tsx`, etc.)
- **Files modified:** `src/renderer/src/components/ui/scroll-area.tsx`, `package.json`, `package-lock.json`
- **Commit:** 86cf933

**2. [Rule 1 - Bug] contextBuilder.test.ts Message fixtures missing sessionId field**
- **Found during:** Task 1 TypeScript verification
- **Issue:** After adding `sessionId` to the messages schema, existing test Message object literals in `contextBuilder.test.ts` failed TypeScript strict check (missing required `sessionId: string | null` field)
- **Fix:** Added `sessionId: null` to the two Message fixture objects in the `getLastN` mock return value
- **Files modified:** `src/main/ai/contextBuilder.test.ts`
- **Commit:** 86cf933
