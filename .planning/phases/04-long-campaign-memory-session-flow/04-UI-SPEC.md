**Phase 4-specific exceptions:**

- Session Start modal dialog width: `max-w-[480px]` — same as Phase 1 Create Campaign modal (three simple free-form text fields, no two-column layout needed).
- End-of-Session Recap modal dialog width: `max-w-[560px]` — same as Phase 3 wizard modal; wider to accommodate the streaming recap textarea and player notes field comfortably.
- Both modals: `max-h-[85vh] overflow-hidden flex flex-col` — consistent with Phase 3 dialog pattern.
- Journal card collapsed state: `px-4 py-3` (16px horizontal, 12px vertical = 3×4 — justified exception, sits between sm and md; keeps the card height compact in a dense timeline).
- Journal card expanded area: `px-4 py-4` (16px all sides — standard md token).
- L1 overflow warning bar: `px-4 py-2` (16px horizontal, 8px vertical = sm). Thin bar, not a full-card alert.
- "Start Session" / "End Session" header button: `h-8` (32px) touch target — consistent with Phase 3 gear icon button.
- Locked chat input banner: `py-2 px-3` (8px vertical, 12px horizontal = 3×4 — inherited from ChatInputArea's existing `p-3` interior).
- Recap textarea min-height: `min-h-[160px]` (160px = 40×4) — needs to show at least 8 lines of recap prose while streaming.
- Player notes textarea in recap modal: `min-h-[80px]` (80px = 20×4) — smaller secondary field.

- **"End Session" button (active session):** `Button variant="outline" size="sm" className="border-destructive/60 text-destructive hover:bg-destructive/10 hover:border-destructive"` — muted brick-red border and label, strengthens on hover. This makes the destructive nature visible without an aggressive red fill at rest.
- **"Begin Session" button (modal CTA):** `Button variant="default" size="sm"` with `--primary` (gold) background — this is the affirmative action that starts the session.
- **"Save Session" button (recap modal CTA):** Same as "Begin Session" — gold primary.
- **Locked chat banner:** `border-t border-border bg-background px-3 py-2` — same container as `ChatInputArea` interior, replacing the textarea + send button when locked. Text: `text-sm text-muted-foreground`.

- **Journal "in progress" placeholder card:** `bg-card/50 border border-border rounded-lg px-4 py-3 opacity-60 cursor-not-allowed` — reduced opacity signals non-interactable.
- **Journal completed session card (collapsed):** `bg-card border border-border rounded-lg px-4 py-3 cursor-pointer hover:bg-secondary hover:border-border transition-colors` — standard card hover.
- **Journal completed session card (expanded):** The collapsed header remains visible; below it, the expanded body has `bg-secondary rounded-b-lg px-4 py-4 border-x border-b border-border` — the top border of the expanded section uses `border-t border-border mt-0` creating a separator below the header row.

## Screen & Component Specifications

### S1. Campaign Header — "Start Session" / "End Session" Button

**Location:** `CampaignViewScreen.tsx` action bar — the `border-b border-border bg-card` strip that holds Back, AI Settings, and Delete. The session button is placed between "AI Settings" and "Delete Campaign":

```
[← Campaigns]     [spacer]     [⚙ AI Settings]  [Start Session]  [🗑 Delete Campaign]
```

When a session is active, "Start Session" transforms to "End Session" in the same position.

**"Start Session" button spec:**
- Component: `<Button variant="outline" size="sm" className="gap-1.5" onClick={openSessionStartModal}>`
- Icon: `<Play className="h-4 w-4" />` (lucide-react `Play`) — signals "begin"
- Label: `"Start Session"` — visible text, not icon-only
- Color: neutral outline — `border-border text-foreground hover:bg-secondary` (no gold accent; session management is not the same as a primary CTA)
- Tooltip: `<Tooltip content="Begin a new play session" delayDuration={600} />`
- Disabled condition: none (button is always available; clicking while a summarization is pending shows a loading state instead)

**"End Session" button spec:**
- Component: `<Button variant="outline" size="sm" className="gap-1.5 border-destructive/60 text-destructive hover:bg-destructive/10 hover:border-destructive" onClick={openEndSessionModal}>`
- Icon: `<Square className="h-4 w-4" />` (lucide-react `Square` — "stop") — signals "end"
- Label: `"End Session"` — visible text
- Color: muted destructive at rest, full destructive on hover — signals consequential action without being aggressive at rest
- Tooltip: `<Tooltip content="End this session and save a recap" delayDuration={600} />`

**Transition between states:**
- No animation between Start/End — the label and icon swap instantly when session state changes (Zustand `isSessionActive` → `true` on `session.start` mutation success). The button is in the same DOM position; no layout shift.
- During the `session.start` mutation in-flight: button shows `<Loader2 className="h-4 w-4 animate-spin" />` + `"Starting…"` label, disabled. Duration: typically < 500ms (DB write + AI narration trigger).
- During `session.end` mutation in-flight: button shows spinner + `"Ending…"` label, disabled.

- Placeholder: `"Anything the DM should know before you start — ongoing effects, NPC relationships, loose threads."`
- Optional: no validation required
- Max length: 1000 characters (no visible counter)
- Helper text: omitted

*All three fields:* `bg-secondary border border-border rounded-md text-sm text-foreground placeholder:text-muted-foreground focus:ring-1 focus:ring-primary focus:border-primary` — consistent with Phase 3 input/textarea styling.

**Footer:**
- `[Not Yet]` — `Button variant="ghost"` — closes modal, no session created; label communicates "session remains not-yet-started"
- `[Begin Session]` — `Button variant="default"` (gold) — primary CTA
  - Never disabled (all fields optional)
  - Click: immediately closes modal + triggers `session.start` mutation + AI auto-narrates opening (D-04)
  - Loading state: button shows spinner + `"Starting…"` after click, modal stays open until mutation succeeds, then closes
  - On success: modal closes, chat locked state lifts, AI narration streams immediately

**Focus order on open:**
1. Current Location input (autofocus on dialog open)
2. Session Goal input
3. Context Notes textarea
4. Not Yet button
5. Begin Session button

**Keyboard:**
- `Escape`: close modal (no confirmation needed — no data loss; fields are optional prompts)
- `Tab` / `Shift+Tab`: standard focus cycle
- `Enter` in Location or Goal (single-line inputs): advance to next field (do NOT submit — form has 3 fields, Enter should tab-forward)
- `Enter` in Context Notes (textarea): insert newline
- `Enter` on "Begin Session" button: submit

**File:** `src/renderer/src/components/ChatInputArea.tsx` (extended with session-awareness)

When `isSessionActive === false` (no session started), the textarea and Send button are replaced by a locked state banner. This extends the existing "no AI config" disabled pattern from Phase 3.

**Locked state layout:**
```
┌──────────────────────────────────────────────────────────────┐
│  (border-t border-border — same as ChatInputArea container)  │
│                                                              │
│  ▶  Start your session to begin playing.                     │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

**Locked state spec:**
- Container: `border-t border-border bg-background px-4 py-4 flex items-center gap-2` — same border-top as ChatInputArea, same background, but taller than the `p-3` input row to give the banner text room

### S4. L1 Context Overflow Warning

**File:** `src/renderer/src/components/StoryScrollPanel.tsx` (inline warning rendered inside the scroll area)

**Trigger:** ContextBuilder v2 emits an `L1_OVERFLOW` signal when the current session's messages exceed the L1 overflow threshold (6000 tokens → falls back to last 30 messages). The renderer receives this as a Zustand flag `isL1Overflow: boolean` via IPC.

**Placement:** A thin bar rendered as a sticky element at the bottom of the `StoryScrollPanel` scroll area — above the `border-t` that separates the scroll from `ChatInputArea`. It does NOT scroll with the message list; it is always visible when active.

Implementation: rendered as a sibling to the scrollable message list inside the scroll panel container, using `flex flex-col h-full`:
```
<div className="flex flex-col h-full overflow-hidden">
  <div className="flex-1 overflow-y-auto">   ← message list
    {messages}
  </div>
  {isL1Overflow && <L1OverflowWarning />}    ← warning bar, always at bottom of scroll area
</div>
```

**L1 Overflow Warning component spec:**
```
┌──────────────────────────────────────────────────────────────────────────┐
│ ⚠  Earlier conversation history is not visible to the AI —              │
│    session context grew beyond model limit.                              │
└──────────────────────────────────────────────────────────────────────────┘
```

- Container: `bg-amber-950/30 border-y border-amber-900/40 px-4 py-2 flex items-start gap-2 shrink-0`
- Icon: `<AlertTriangle className="h-4 w-4 text-amber-500 shrink-0 mt-0.5" />` (lucide-react)
- Text: `<span className="text-sm text-amber-400">Earlier conversation history is not visible to the AI — session context grew beyond model limit.</span>` — single line if panel is wide enough; wraps gracefully
- The bar has no close/dismiss button — it persists as long as `isL1Overflow` is `true` (which is for the rest of the session once triggered)
- This is NOT a modal, NOT a toast, NOT a dialog — only a thin informational bar at the base of the scroll area
- No animation on appear — renders immediately when condition is detected

```
│                                                               │

│                                                               │
│  Session Recap                                                │
│  ┌────────────────────────────────────────────────────────┐  │
│  │ The party descended into the ancient crypt, where      │  │

│  │ (streaming prose — same font as story scroll)          │  │
│  └────────────────────────────────────────────────────────┘  │
│                                                               │

│  [__________________________________________________________] │

│                                                               │

└───────────────────────────────────────────────────────────────┘
```

**Session Recap area — Phase A (streaming):**
- Label: `"Session Recap"` (14px / 600) above the textarea
- During streaming: a read-only `<div>` styled to look identical to the `StoryScrollPanel` prose — `text-sm leading-[1.6] text-foreground bg-secondary border border-border rounded-md px-4 py-4 min-h-[160px] overflow-y-auto`
- Streaming cursor: same `|` blink animation as Phase 3 story scroll — `<span aria-hidden="true" className="animate-[blink_1s_ease-in-out_infinite] text-foreground">|</span>`
- No scrollbar shown until content overflows — `overflow-y-auto`
- While streaming: `[Save Session]` button is disabled with `"Generating recap…"` label

**Session Recap area — Phase B (editable after streaming completes):**
- The streaming `<div>` transitions to a `<Textarea>` with the same visual appearance — `text-sm leading-[1.6] text-foreground bg-secondary border border-border rounded-md px-4 py-4 min-h-[160px] resize-y`
- The textarea is pre-populated with the full streamed recap text
- `resize-y` (not `resize-none`) — player may want to expand the recap textarea to read/edit more comfortably
- `[Save Session]` button becomes enabled once streaming completes
- Player can freely edit the recap text (the edited version is what gets saved to the DB and used for Layer 2 memory injection — D-10)
- Helper text below textarea: `"You can edit the recap — the saved version becomes part of the AI's memory for future sessions."` (14px / 400 / `--muted-foreground`)

**Player Notes field (always present, editable from dialog open):**
- Label: `"Player Notes (optional)"` (14px / 600)
- Component: `<Textarea rows={3} resize-none />` — `min-h-[80px]`
- Placeholder: `"Personal notes — your own observations, theories, reminders."`
- Available to type in during streaming (while recap is generating) — no reason to lock it
- Helper text: `"Visible only to you — not injected into the AI's context."` (14px / 400 / `--muted-foreground`)

**Footer:**
- `[Keep Playing]` — `Button variant="ghost"` — closes modal WITHOUT saving the session; label communicates that the session remains active and the player can continue. No destructive confirmation needed (the session is still active in the DB — it was just not ended). The "End Session" button reappears in the header if user dismisses.
- `[Save Session]` — `Button variant="default"` (gold) — disabled until streaming completes, then enabled
  - Loading state while saving: `"Saving…"` + spinner, disabled
  - On success: modal closes; session is saved; rolling summary generation triggered in background; chat locked state activates; header reverts to "Start Session" button
  - On error: modal stays open; inline error below footer: `"Could not save the session. Try again."` (14px / `--destructive`)

**Keyboard:**
- `Escape`: close modal (no save) — same as Keep Playing
- `Tab` / `Shift+Tab`: focus cycles through recap textarea → player notes → Keep Playing → Save Session
- `Ctrl+Enter`: save session (fires "Save Session" if button is enabled) — `react-hotkeys-hook` binding within the modal

**Container:**
- `TabsContent value="journal"` → `<div className="h-full flex flex-col overflow-hidden p-0">`
- The timeline is a `<ScrollArea className="flex-1">` containing the stacked cards with `className="flex flex-col gap-2 p-4"`
- Empty state (no sessions yet): `"No sessions yet. Start a session to begin journaling."` — centered, 14px / 400 / `--muted-foreground`, vertically centered in the scroll area

**"Session N in progress" placeholder card:**
- Rendered at the TOP of the list if `isSessionActive === true`
- Component: a non-interactive card (no `Collapsible` — it cannot expand)
- Visual: `bg-card/50 border border-border rounded-lg px-4 py-3 flex items-center gap-2 opacity-60 cursor-not-allowed`
- Icon: `<Circle className="h-3 w-3 text-amber-500 animate-pulse" />` (small pulsing dot to suggest "live")
- Text: `<span className="text-sm italic text-muted-foreground">Session {N} in progress</span>` — N is `maxSessionNumber + 1` derived from the sessions query
- No chevron, no expand, no click affordance

**Completed session card (collapsed state):**
- Component: shadcn `Collapsible` wrapper with a trigger row
- Container: `<div className="bg-card border border-border rounded-lg overflow-hidden">`
- Trigger row: `<CollapsibleTrigger className="w-full px-4 py-3 flex items-center gap-3 hover:bg-secondary cursor-pointer transition-colors">`
  - Chevron: `<ChevronDown className="h-4 w-4 text-muted-foreground transition-transform duration-150 data-[state=open]:rotate-180" />` — rotates 180° when expanded (Radix data attribute)
  - Session number: `<span className="text-sm font-semibold text-foreground">Session {N}</span>`
  - Date: `<span className="text-sm text-muted-foreground">{formattedDate}</span>` — formatted as `"MMM D, YYYY"` via dayjs (e.g., "May 24, 2026")
  - Location: `<span className="text-sm italic text-muted-foreground truncate flex-1">— {location}</span>` — italic, truncated with ellipsis if long; omitted entirely if location is null/empty (no `—` shown)
  - Layout: `flex items-center gap-3` — chevron, session number, date, then location taking remaining space

**Completed session card (expanded state):**
- `<CollapsibleContent>` with `className="border-t border-border"`
- Expanded body: `<div className="px-4 py-4 flex flex-col gap-4 bg-secondary">`

  *AI Recap section:*
  - Label: `<h3 className="text-sm font-semibold text-foreground">Session Recap</h3>`
  - Content: `<div className="text-sm leading-[1.6] text-foreground">` — renders the `ai_recap` text. Plain text, no `react-markdown` rendering (recap is already formatted prose from the AI; no markdown expected). Read-only — no click-to-edit, no edit button. Styled exactly like `StoryScrollPanel` prose (same font, size, line-height) for consistent narrative feel.

  *Player Notes section:*
  - Label: `<h3 className="text-sm font-semibold text-foreground">Player Notes</h3>`
  - State: always editable (D-24)
  - Component: `<Textarea className="min-h-[64px] bg-background border-border text-sm text-foreground placeholder:text-muted-foreground resize-y" />` — `bg-background` (darker than the expanded card's `bg-secondary` surface, creating a subtle inset)
  - Placeholder: `"Click to add personal notes…"` (shown when `player_notes` is null/empty)
  - Edit behavior: textarea is always focusable/editable; player clicks into it to start typing
  - Save affordance: Below the textarea, a `<div className="flex items-center justify-between mt-2">`:
    - Left: `<span className="text-[12px] text-muted-foreground">Ctrl+Enter to save</span>` — keyboard hint
    - Right: `<button className="text-sm text-primary underline hover:no-underline">Save Notes</button>` — link-style button, always visible (not only on change)
  - On `Ctrl+Enter` OR click "Save Notes": fires `sessions.updatePlayerNotes.mutate({ sessionId, notes })` — `react-hotkeys-hook` binding scoped to the expanded card's textarea
  - Loading state: "Save Notes" link text becomes `"Saving…"` (14px / `--muted-foreground`, no underline) for < 500ms, then reverts
  - No explicit "saved" success state beyond the button returning to normal (silent, consistent with Phase 3's "silent close on success" pattern)
  - Error state: if mutation fails, `"Could not save notes. Try again."` appears in 14px / `--destructive` below the textarea, persisting until the player dismisses or successfully saves (not auto-dismissed — the player needs to know the save failed and act on it)

**Session count and ordering:**
- Sessions ordered newest-first (`ORDER BY session_number DESC`)
- Session numbers displayed as `"Session 1"`, `"Session 2"`, etc. (D-22) — monotonically from 1
- No pagination in Phase 4 — full scrollable list (ScrollArea handles overflow for 20-30 sessions)

## Interaction Contract

### Session Start Flow

| Action | Trigger | Result |
|--------|---------|--------|
| Open Session Start modal | Click "Start Session" in header | Dialog opens, Current Location autofocused |
| Pre-fill location | Modal opens on session 2+ | Current Location field populated with last session's location |
| Navigate fields | `Tab` / `Shift+Tab` | Standard focus cycle through 3 fields + 2 buttons |
| Submit | Click "Begin Session" OR `Enter` on button | Modal closes → session created → AI narration starts immediately |
| Dismiss | Click "Not Yet" OR `Escape` | Modal closes, no session created, header button unchanged |
| Prevent close during submission | Mutation in-flight | Modal stays open, button shows spinner; `Escape` ignored during flight |

### Chat Input Locked State

| Action | Trigger | Result |
|--------|---------|--------|
| See locked banner | No active session | Textarea + Send replaced by locked banner |
| Click locked banner area | Any click in locked area | No-op — no interaction afforded |
| Session starts | `isSessionActive` → `true` | Locked banner replaced by textarea + Send; textarea autofocused |

### End-of-Session Recap Flow

| Action | Trigger | Result |
|--------|---------|--------|
| Open End Session modal | Click "End Session" in header | Dialog opens, recap streaming begins immediately |
| Edit player notes | Click Player Notes textarea | Edit freely (editable from dialog open) |
| Edit recap | After streaming completes | Recap textarea becomes editable; player can revise |
| Save session | Click "Save Session" OR `Ctrl+Enter` | Saves session, closes modal, chat locks, header reverts |
| Dismiss | Click "Keep Playing" OR `Escape` | Modal closes, session remains active (not ended) |
| Prevent save during streaming | Recap still streaming | "Save Session" button disabled + `"Generating recap…"` label |

### Session Journal Tab

| Action | Trigger | Result |
|--------|---------|--------|
| Switch to Journal tab | Click "Journal" tab | `SessionJournalTab` renders with timeline |
| Expand session card | Click collapsed card / trigger row | `CollapsibleContent` opens; recap + notes visible |
| Collapse session card | Click expanded card trigger | `CollapsibleContent` closes |
| Edit player notes | Click into notes textarea (expanded) | Textarea focused; existing notes pre-populated |
| Save notes (pointer) | Click "Save Notes" link | `sessions.updatePlayerNotes` mutation fires |
| Save notes (keyboard) | `Ctrl+Enter` in notes textarea | Same mutation fires |
| Multiple cards expanded | Open multiple cards | Multiple cards can be expanded simultaneously (no accordion behavior — player may want to compare sessions) |

### Keyboard Accessibility

| Surface | Keys | Behavior |
|---------|------|----------|
| Session Start modal | `Tab` / `Shift+Tab` | Move through Location → Goal → Context Notes → Not Yet → Begin Session |
| Session Start modal | `Escape` | Close modal (no save) |
| Session Start modal | `Enter` in single-line inputs | Tab-forward to next field (not submit) |
| End Session modal | `Ctrl+Enter` | Save session (when "Save Session" is enabled) |
| End Session modal | `Escape` | Close modal (no save) |
| Journal tab | `Tab` | Move through collapsible triggers in DOM order |
| Journal collapsed card | `Enter` / `Space` | Toggle expand/collapse |
| Journal player notes textarea | `Ctrl+Enter` | Save notes |
| Journal "Save Notes" link | `Enter` / `Space` | Save notes |
| "Start Session" button | `Enter` / `Space` | Open Session Start modal |
| "End Session" button | `Enter` / `Space` | Open End Session modal |

## Accessibility

| Surface | Requirement | Implementation |
|---------|-------------|----------------|
| Session Start modal | Focus trap | Radix Dialog handles automatically |
| Session Start modal | Role | `<dialog>` via Radix — `role="dialog" aria-modal="true"` |
| Session Start modal | Title | `<DialogTitle>Start Session</DialogTitle>` — announced on open |
| End Session modal | Focus trap | Radix Dialog handles automatically |
| End Session modal | Streaming recap | Wrap the streaming `<div>` in `<div aria-live="polite" aria-atomic="false">` — tokens announced progressively. Same pattern as `StoryScrollPanel`. |
| Streaming cursor | Hidden from AT | `<span aria-hidden="true">|</span>` — not announced |
| Locked chat banner | Role | `<div role="status" aria-live="polite">` — announces to screen readers when state changes |
| L1 overflow warning | Alert role | `<div role="alert" aria-live="assertive">` — announced immediately when condition triggers |
| L1 warning icon | Screen reader text | `<span className="sr-only">Context window warning:</span>` preceding the visible text |
| "Start Session" button | Descriptive | `aria-label="Start Session"` — matches visible label; tooltip via `aria-describedby` |
| "End Session" button | Destructive signal | `aria-label="End Session — ends this session and opens recap"` — more context than label alone |
| Journal tab | Selected state | Radix `Tabs` sets `aria-selected="true"` on the active `TabsTrigger` |
| Journal collapsed card | Expanded state | `<CollapsibleTrigger aria-expanded={isOpen}>` — Radix handles |
| Journal notes textarea | Label | `<Label htmlFor={notesId}>Player Notes</Label>` with matching `id` on textarea |
| "Save Notes" link button | Role | `<button type="button">Save Notes</button>` — not `<a>` without `href` |
| "Session in progress" card | Non-interactive signal | `aria-disabled="true"` on the card container; `role="status"` to announce the in-progress state |
