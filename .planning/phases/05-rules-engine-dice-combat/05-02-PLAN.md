<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| AI response → DB mutations | Tool-call args (or JSON-tail block) are AI-generated and untrusted; they cross into SQLite writes |
| JSON-tail text → mutation pipeline | Embedded fenced block parsed from model output may inject extra/malformed mutations |
| renderer → main | Renderer sends the chat message but NEVER originates tool calls; tool calls only come from AI response processing in main |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-02-01 | Tampering | applyMutationBatch tool-call ingestion | mitigate | Zod `.safeParse()` on every tool call before applying; numeric fields bounded `.int().min(0).max(...)` (negative HP_max / infinite XP rejected); invalid calls logged + dropped (D-06) |
| T-05-02-02 | Tampering | stripAndParseJsonTail | mitigate | JSON-tail only parsed when no native tool calls present (D-02 fallback); parsed mutations run through the SAME safeParse pipeline; regex end-anchored to `$` so mid-text code blocks are not treated as mutation blocks (Pitfall 3) |
| T-05-02-03 | Tampering | renderer forging tool results over IPC | mitigate | Tool calls originate only from streamText onFinish in main process; the ai:send-message handler never accepts mutation payloads from the renderer |
| T-05-02-04 | Denial of Service | runaway/double mutation application | mitigate | D-04 batch-in-onFinish (no execute on tools, Pitfall 1); native-call flag prevents JSON-tail double-apply; single db.transaction() bounds the write |
| T-05-02-SC | Tampering | npm/pip/cargo installs | mitigate | No package installs in this plan (rpg-dice-roller installed in 05-01); no new dependencies |
</threat_model>
