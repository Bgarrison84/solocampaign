## Spacing Scale

Inherited from Phase 1, unchanged. Multiples of 4 only.

| Token | Value | Usage in Phase 5 |

| xs | 4px | Initiative order number to combatant name gap; die button icon-label gap; condition badge gaps in combat rows |
| sm | 8px | Combatant row interior padding (vertical); spell row interior padding; level-up modal field row gaps; HP stepper +/- button size |
| md | 16px | Combat tracker tab padding; spell section padding; level-up modal body padding; rest picker dialog body padding |
| lg | 24px | Level-up modal outer padding (DialogContent); rest picker outer padding |
| xl | 32px | (available from prior phases — not added in Phase 5) |
| 2xl | 48px | (not used in Phase 5) |
| 3xl | 64px | (reserved, not used) |

**Phase 5-specific exceptions:**

- Combatant row collapsed height: `py-2 px-3` (8px vertical, 12px horizontal = 3×4 — compact list; keeps many combatants visible without scrolling).
- Combatant row expanded area: `px-4 py-3` (16px horizontal, 12px vertical).
- Dice roller popover body: `p-3` (12px = 3×4 — tight grid of die buttons).
- Die button size: `h-9 w-9` (36px = 9×4) — square touch targets for the 7 die-type buttons.
- HP bar in combat tracker row: `h-1.5` (6px = 1.5×4 — ultra-thin progress bar, purely informational beside the HP fraction text).
- showDiceRoll chip in StoryScrollPanel: `px-3 py-2` (12px horizontal, 8px vertical).
- Toast chip: `px-3 py-1.5` (12px horizontal, 6px vertical = 1.5×4).
- Level-up banner: `px-4 py-3` (16px horizontal, 12px vertical = 3×4) — same cadence as Phase 4 journal card collapsed state.
- Level-up modal width: `max-w-[480px]` — same as Session Start modal (Phase 4).
- Rest picker dialog width: `max-w-[400px]` — narrower than level-up; two-button choice only.
- Short rest hit die modal width: `max-w-[440px]`.
- Spell list section spell card expanded area: `px-4 py-3` (16px horizontal, 12px vertical).
- "Add Combatant" form: `p-3 gap-3` (12px padding, 12px gap).

## Screen & Component Specifications

### S1. Campaign Header — "Start Combat" + "Rest" Buttons

**Location:** `CampaignViewScreen.tsx` action bar — the `border-b border-border bg-card` strip.

**Updated header layout (left → right):**
```
[← Campaigns]  [spacer]  [⚙ AI Settings]  [Start Combat]  [Rest]  [Start Session]  [🗑 Delete Campaign]
```

When combat is active, "Start Combat" becomes "End Combat" in the same position:
```
[← Campaigns]  [spacer]  [⚙ AI Settings]  [End Combat]  [Rest]  [End Session]  [🗑 Delete Campaign]
```

**"Start Combat" button spec:**
- Component: `<Button variant="outline" size="sm" className="gap-1.5" onClick={handleStartCombat}>`
- Icon: `<Swords className="h-4 w-4" />` (lucide-react `Swords`)
- Label: `"Start Combat"` — visible text
- Color: neutral outline — `border-border text-foreground hover:bg-secondary`
- Tooltip: `<Tooltip content="Begin a combat encounter" delayDuration={600} />`
- onClick: sets `combatStore.isCombatActive = true`, sets `activeTab = 'combat-tracker'` in campaignViewStore (auto-switches panel tab)
- Disabled condition: none — always clickable; clicking while already in combat shows "End Combat" instead

**"End Combat" button spec:**
- Component: `<Button variant="outline" size="sm" className="gap-1.5 border-destructive/60 text-destructive hover:bg-destructive/10 hover:border-destructive" onClick={handleEndCombat}>`
- Icon: `<Swords className="h-4 w-4" />` (same icon)
- Label: `"End Combat"` — visible text
- Color: muted destructive at rest, full destructive on hover
- Tooltip: `<Tooltip content="End the current encounter" delayDuration={600} />`
- onClick: calls `combat.endCombat` mutation — marks all combatants `is_active = false`; writes `combat_ended` event; shows mutation chip `"Combat ended"`

**"Rest" button spec:**
- Component: `<Button variant="outline" size="sm" className="gap-1.5" onClick={() => setShowRestPicker(true)}>`
- Icon: `<Moon className="h-4 w-4" />` (lucide-react `Moon`)
- Label: `"Rest"` — visible text
- Color: neutral outline — `border-border text-foreground hover:bg-secondary`
- Tooltip: `<Tooltip content="Take a short or long rest" delayDuration={600} />`

**State management:**
- New Zustand store: `combatStore` with `isCombatActive: boolean`, `activeCombatId: string | null`, `currentTurnOrder: number`
- New Zustand atom on `campaignViewStore` (or `useCampaignViewStore`): `activeTab: string` — when Start Combat fires, set `activeTab = 'combat-tracker'`; pass this value as `value` on the `<Tabs>` component (controlled tabs).

```jsx

    onKeyDown={e => { if (e.key === 'Enter') handleExpressionRoll() }}
    aria-label="Dice expression"
  />

</div>
<p className="text-[11px] text-muted-foreground mt-1">e.g. 4d6kh3, d20+5</p>
```

**Roll result format:**
- Single die click: `[d20: 14] ` — prepended to textarea value
- Expression: `[2d6+3: 11] ` — expression + result + space
- The space after ensures the player's next-typed character starts naturally
- Result is inserted at the START of the textarea value (not end): `setTextareaValue('[' + dieType + ': ' + result + '] ' + existingValue)`
- If textarea already has a roll prefix from a prior roll, replace it: regex `/^\[[\w\d+\-*\/]+: \d+\] /` strips old prefix before adding new one

**Library:** Use `rpg-dice-roller` package (evaluate at plan time; `dice-typescript` is the fallback). Both support `4d6kh3` notation. Roll evaluation runs in the renderer process (no IPC needed for display-only rolling).

**State transitions:**

| Action | Trigger | Result |
|--------|---------|--------|
| Open roller | Click dice button | Popover opens above button |
| Quick roll (die button) | Click d20 etc. | Roll computed; `[d20: 14] ` prepended to textarea; popover closes; textarea focused |
| Expression roll | Click Roll or Enter in input | Expression evaluated; `[2d6+3: 11] ` prepended; popover closes; textarea focused |
| Invalid expression | Bad notation in input | Roll button disabled; `<span className="text-[11px] text-destructive">Invalid expression</span>` shown below input |
| Player deletes prefix | Backspace in textarea | Prefix removed; no dice event logged (D-21 — logging only on message send) |
| Send with prefix | Player clicks Send | Message sends with `[d20: 14] I attack the goblin!`; `dice_roll` campaign_events row written with `{expression: 'd20', result: 14, breakdown: [14]}` |

|--------|---------|--------|
| Start combat | Click "Start Combat" in header | `isCombatActive = true`; auto-switch to Combat tab; empty state replaced by initiative list |
| AI adds combatant | `addCombatant` tool call | New row appears at correct initiative_order position; toast chip "Goblin added" |

| View combatant details | Click any row | Row expands: HP stepper + condition picker |
| Adjust HP | Click + or − on Stepper in expanded row | Optimistic update; tRPC mutation; HP bar updates |
| Toggle condition | Click ConditionBadge in expanded row | Optimistic update; badge highlights; tRPC mutation |
| AI updates HP | `updateHp` tool call | Affected row HP updates; toast chip "−6 HP" |
| AI applies condition | `applyCondition` tool call | Condition badge highlighted on affected row; toast chip "Poisoned applied" |
| AI advances turn | `currentTurnOrder` state changes | Active row highlight shifts to next combatant in order |
| End combat (player) | Click "End Combat" in header | All rows cleared; empty state; toast chip "Combat ended"; tab stays on Combat |
| End combat (AI) | `endCombat` tool call | Same as above |

| Action | Trigger | Result |
|--------|---------|--------|
