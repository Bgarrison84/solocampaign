## Spacing Scale

Inherited from Phase 1, unchanged. Multiples of 4 only.

| Token | Value | Usage in Phase 6 |
|-------|-------|-----------------|
| xs | 4px | Badge-to-name gap in NPC/quest rows; time-of-day icon-to-text gap; breadcrumb segment separator gap |
| sm | 8px | NPC row vertical interior padding; quest row vertical interior padding; faction row vertical padding; WorldStateBar vertical padding |
| md | 16px | QuestsTab and NpcTrackerTab container padding; WorldStateBar horizontal padding |
| lg | 24px | (available from prior phases — not added in Phase 6) |
| xl | 32px | (available from prior phases — not added in Phase 6) |
| 2xl | 48px | (not used in Phase 6) |
| 3xl | 64px | (reserved, not used) |

**Phase 6-specific exceptions:**

- WorldStateBar height: `py-2 px-4` (8px vertical, 16px horizontal) — slim information bar; same vertical rhythm as the L1 Overflow warning bar from Phase 4.
- NPC row: `px-3 py-2` (12px horizontal, 8px vertical = 3×4 on horizontal — inherited Phase 5 combatant-row cadence for dense lists).
- Quest row: `px-3 py-2` — same as NPC row for visual consistency between the two new tabs.
- Faction collapsible trigger row: `px-3 py-2 mt-4` — 16px top margin (4×4) to separate the Factions section from the NPC list above it.
- Faction row interior: `px-3 py-1.5` (12px horizontal, 6px vertical = 1.5×4). Exception: `py-1.5` is the single permitted exception to the 4-point grid for this element only — faction tier rows are the most compact list item in the app; the alternative `py-1` feels too tight for text readability and `py-2` produces too much height in a faction list that may grow to 8–10 entries. Document the exception explicitly here.
- Icon sizes `h-3.5 w-3.5` on chip icons: inherited Phase 5 permitted icon exception. Applies to chip icons only, not to padding, margin, or gap utilities.

### S1. WorldStateBar — Campaign Header Extension

**Location:** `CampaignViewScreen.tsx` — a new `<div>` rendered BELOW the action bar `<div>` and ABOVE `<PanelGroup>`. Inserted between line 452 and line 454 in the current file.

**Condition to render:** Only rendered when `campaignQuery.data` has at least one non-null world-state field (`worldTimeOfDay` or `worldLocationPath`). If both are null (AI has not yet called `updateWorldTime` or `updateLocation`), the bar is absent — no empty placeholder.

**Data source:** The existing `campaignQuery` (`trpc.campaigns.get.query({ id })`) — Phase 6 adds nullable world-state columns to the `campaigns` table (migration 0006). No separate query needed.

**WorldStateBar layout:**
```
┌──────────────────────────────────────────────────────────────────────┐
│ [🌙] Evening • Day 14 • Autumn    [flex-1 spacer]    📍 Forest > ... │
└──────────────────────────────────────────────────────────────────────┘
```

**Exact structure and className:**
```jsx
{(campaign.worldTimeOfDay || campaign.worldLocationPath) && (
  <div className="flex items-center gap-4 px-4 py-2 border-b border-border bg-background text-xs text-muted-foreground shrink-0">
    {/* Left: Calendar */}
    {campaign.worldTimeOfDay && (
      <div className="flex items-center gap-1.5 shrink-0">
        <span aria-hidden="true">{timeOfDayIcon(campaign.worldTimeOfDay)}</span>
        <span>
          {campaign.worldTimeOfDay}
          {campaign.worldDayNumber != null && ` • Day ${campaign.worldDayNumber}`}
          {campaign.worldSeason && ` • ${campaign.worldSeason}`}
        </span>
      </div>
    )}
    {/* Spacer */}
    <span className="flex-1" />
    {/* Right: Location breadcrumb */}
    {campaign.worldLocationPath && (
      <div className="flex items-center gap-1 min-w-0">
        <span aria-hidden="true" className="shrink-0 text-muted-foreground/60">📍</span>
        <span
          className="truncate max-w-[280px]"
          title={formatLocationPath(campaign.worldLocationPath)}
          aria-label={`Current location: ${formatLocationPath(campaign.worldLocationPath)}`}
        >
          {formatLocationPath(campaign.worldLocationPath)}
        </span>
      </div>
    )}
  </div>
)}
```

**`timeOfDayIcon` helper:**
```typescript
function timeOfDayIcon(timeOfDay: string): string {
  switch (timeOfDay) {
    case 'Morning':   return '☀️'
    case 'Afternoon': return '🌤'
    case 'Evening':   return '🌙'
    case 'Night':     return '🌑'
    default:          return '🕐'
  }
}
```

Note: emojis used here are aria-hidden and serve only as visual glyphs. CONTEXT.md D-12 specifies a time-of-day icon; emoji is the approved choice (no additional icon library dependency, consistent with the dice roller `🎲` emoji from Phase 5).

**`formatLocationPath` helper:**
```typescript
function formatLocationPath(jsonPath: string | null): string {
  if (!jsonPath) return ''
  try {
    const parts = JSON.parse(jsonPath) as string[]
    return parts.join(' > ')
  } catch {
    return jsonPath // fallback if not JSON
  }
}
```

**Updated campaign header layout (top → bottom):**
```
Row 1 (action bar):  [← Campaigns]  [spacer]  [AI Settings]  [Start Combat]  [Rest]  [Start/End Session]  [Delete]
Row 2 (WorldStateBar, conditional): [☀️ Morning • Day 3 • Spring]  [spacer]  [📍 Phandalin > Tresendar Manor]
Row 3 (PanelGroup — left/right panels)
```

**State transitions:**

| State | Condition | Result |
|-------|-----------|--------|
| Bar absent | `worldTimeOfDay` and `worldLocationPath` both null | No row rendered; campaign view goes directly from action bar to panels |
| Bar appears (partial) | AI calls `updateWorldTime` (but not `updateLocation`) | Only left side (calendar) renders; right side absent; `campaigns.get` cache invalidated |
| Bar appears (partial) | AI calls `updateLocation` (but not `updateWorldTime`) | Only right side (breadcrumb) renders; left side absent |
| Bar fully populated | Both tool calls have fired at least once | Both left and right sides visible |
| Calendar updates | AI calls `updateWorldTime` again | `campaignQuery` refetches; bar updates inline — no animation |
| Location updates | AI calls `updateLocation` again | Same — refetch, inline update |

```jsx
    </div>
  </ScrollArea>
</div>
```

```typescript
function npcRelationshipColor(relationship: string): string {
  switch (relationship) {
    case 'Friendly': return 'text-green-400 border-green-400'
    case 'Hostile':  return 'text-red-400 border-red-400'
    case 'Neutral':  return 'text-muted-foreground border-border'
    case 'Unknown':  return 'text-muted-foreground border-border'
    default:         return 'text-muted-foreground border-border'
  }
}
```

```typescript
function factionTierColor(tier: string): string {
  switch (tier) {
    case 'Hostile':    return 'text-red-400 border-red-400'
    case 'Unfriendly': return 'text-orange-400 border-orange-400'
```

## Interaction Contract

### Quest Log Flows

| Action | Trigger | Result |
|--------|---------|--------|
| View quest log | Click "Quests" tab | `QuestsTab` renders; quests listed chronologically |
| Quest added | AI calls `addQuest` | `quest_added` chip appears (green scroll icon, "Quest: {name}"); `quests.list` cache invalidated; new row at bottom |
| Quest completed | AI calls `updateQuestStatus({ status: 'Completed' })` | `quest_complete` chip appears (muted checkmark, "Quest complete!"); badge on that row changes from Active green → Completed muted |
| Quest failed | AI calls `updateQuestStatus({ status: 'Failed' })` | No chip (D-05); badge on that row changes from Active green → Failed red; silent |
| Empty tab | No quests yet | Empty state with `ScrollText` icon + body text |

### NPC Tracker Flows

| Action | Trigger | Result |
|--------|---------|--------|
| View NPC tracker | Click "NPC Tracker" tab | `NpcTrackerTab` renders; NPC list + Factions section |
| NPC added | AI calls `addNpc` | `npc_added` chip appears (muted UserPlus, "{name} encountered"); `npcs.list` cache invalidated; new row at bottom |
| NPC relationship updated | AI calls `updateNpc` | No chip; `npcs.list` cache invalidated; relationship badge or description updates silently |
| Faction added | AI calls `updateFaction` (new name) | No chip; `factions.list` cache invalidated; new faction row appears in Factions section |
| Faction tier updated | AI calls `updateFaction` (existing name) | No chip; `factions.list` cache invalidated; tier badge updates |
| Expand/collapse factions | Click Factions collapsible trigger | Collapsible animates open/closed (150ms) |
| Empty tab | No NPCs and no factions | Empty state with `Users` icon + body text |

### World State Bar Flows

| Action | Trigger | Result |
|--------|---------|--------|
| Bar absent (initial) | Campaign opens, AI hasn't set world time or location | No bar rendered — header goes directly from action bar to panels |
| Bar appears (time) | AI calls `updateWorldTime` for the first time | `campaigns.get` cache invalidated; WorldStateBar renders with calendar left side |
| Bar appears (location) | AI calls `updateLocation` for the first time | `campaigns.get` cache invalidated; WorldStateBar renders with breadcrumb right side |
| Calendar updates | AI calls `updateWorldTime` again | Bar re-reads from refetched campaign data; time/day/season update inline |
| Location updates | AI calls `updateLocation` with new path | Bar re-reads from refetched campaign data; breadcrumb updates inline |
| Long path truncates | Location path exceeds max-w-[280px] | Text truncates with ellipsis; full path visible on hover via native `title` attribute |

### Inspiration Flow

| Action | Trigger | Result |
|--------|---------|--------|
| Inspiration awarded | AI calls `awardInspiration` | `inspiration` chip appears (amber star, "Inspiration awarded!"); `characters.getByCampaignId` cache invalidated; `hasInspiration` flips on character sheet |
| Inspiration visible | Player opens Character Sheet tab | Inspiration indicator already existed on character sheet; Phase 6 does not change its visual treatment |

### Keyboard Accessibility

| Surface | Keys | Behavior |
|---------|------|----------|
| Quests tab trigger | `Tab` + `Enter`/`Space` | Switch to Quests tab |
| NPC Tracker tab trigger | `Tab` + `Enter`/`Space` | Switch to NPC Tracker tab |
| QuestsTab scroll area | `Tab` + arrow keys | Scroll through quest list via ScrollArea |
| NpcTrackerTab scroll area | `Tab` + arrow keys | Scroll through NPC list and factions |
| Factions collapsible trigger | `Tab` + `Enter`/`Space` | Toggle Factions section open/closed |
| WorldStateBar location text | (passive element — no interaction required) | `aria-label` on breadcrumb span announces full path to screen readers |
