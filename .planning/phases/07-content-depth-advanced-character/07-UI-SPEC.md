## Spacing Scale

Multiples of 4 only. Inherited from globals.css `--radius: 0.5rem` token baseline.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gaps, inline badge padding, stepper button gaps |
| sm | 8px | Compact row spacing, pill chip gaps, within-card padding |
| md | 16px | Default section padding, dialog body padding, card internal padding |
| lg | 24px | Between major sections within a modal or tab pane |
| xl | 32px | Modal max-height offset, library screen panel internal padding |
| 2xl | 48px | Wizard step content area vertical rhythm |
| 3xl | 64px | Full-screen library screen top/bottom breathing room |

Exceptions:
- Touch target minimum: 44px height for all interactive controls (stepper buttons, pill chips, feat picker rows). Matches existing `Stepper` `lg` size (36px) — ensure `h-11` wrapper where native sizing falls below 44px.
- Collapsible section toggle rows: 40px minimum height (existing `Collapsible` pattern).
- Point buy stepper buttons: 32px (matches existing `Stepper` `md` size config `h-7 w-7`).

## Typography

Four roles, two weights. All values match the existing globals.css/shadcn token baseline.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px (text-sm) | 400 (regular) | 1.5 | Paragraph copy, descriptions, flaw text, feat descriptions, world brief content |
| Label | 14px (text-sm) | 600 (semibold) | 1.4 | Section headings within modals, field labels, stat abbreviations |
| Heading | 20px (text-xl) | 600 (semibold) | 1.2 | Modal titles extended (multiclass branch heading, Epic Boon heading), library screen section title |
| Display | 28px | 600 (semibold) | 1.1 | New level number in LevelUpModal (matches existing Phase 5 `text-[28px]`) |

Monospace: `font-mono` class (existing shadcn default) applied to:
- Dice roll results in LevelUpModal HP section (existing pattern).
- Budget counter "X / 27 pts" in point buy (mechanical number).
- Encumbrance "X / Y lbs" running total.

| Muted text | `--muted-foreground` | `oklch(0.60 0.010 260)` | Flaw descriptions, prerequisite text, "✓ used" reroll label, secondary labels, empty state body |

| Red severe | Tailwind `text-red-400` / `bg-red-950/60` | Condition pattern | Encumbrance "Heavily Encumbered" badge (10× STR threshold); remove-companion confirm state |

**Accent (`--accent-gold`) reserved for:**
- Active wizard step circle (WizardProgress, existing pattern)
- Completed wizard step circle fill (existing pattern)
- Method selector active tab indicator (Tabs `data-[state=active]` underline, Tailwind `border-accent-gold`)
- Active party member chip (selected character in switcher pill row)
- Level display number in LevelUpModal (existing pattern)
- ASI/Feat choice selected radio indicator
- Epic Boon list item selected state
- SRD library section pill active state
- Point buy budget counter when budget is exactly met (neutral → gold transition)
- "+ Add Character" button (primary CTA in party mode)

Do not use `--accent-gold` for: feat picker rows (use `--secondary` hover), companion type badges (use existing condition-badge palette), flaw checkbox fills (use `accent-amber-500` as per existing checkbox pattern).

### Components to EXTEND (existing files)

#### `StepAbilityScores.tsx`

**Method selector (new, top of step):**
- Render four `Tabs` (`shadcn/ui tabs`) at the top of the step: "Standard Array" | "4d6 Roll" | "Point Buy" | "Manual".
- Tab trigger height: 36px (`h-9`). Tab bar sits above the existing two-column layout.
- Switching tab fires `resetAbilityScores()` — resets all six scores to `null`. Show a confirmation only if any score is already assigned (inline text: "Switching method will clear your current scores. Continue?" with "Yes, switch" and "Keep current" inline buttons, `h-7 text-xs` size).
- Tab state stored in local `useState<AbilityScoreMethod>` (not `wizardState`; persisted only on "Next" advance).

**Point Buy tab (new content area replacing the two-column layout):**
- Layout: single column, full-width table of 6 ability rows.
- Each row: `[ABBREV label w-10] [− button] [score value w-8 text-center font-mono] [+ button] [modifier w-12 text-center text-sm]`.
- − and + buttons: `Stepper` component `size="md"` (h-7 w-7, existing). Score range: 8–15. Decrement disabled at 8; increment disabled at 15.
- Budget bar (top of table, above rows): `"X / Y pts"` — X = points spent, Y = 27 + flaw bonuses. Both values in `font-mono`. Color: muted when under budget, `text-accent-gold` when exactly at budget, `text-destructive` when over budget (cannot advance). Component layout: `flex justify-between mb-3 text-sm`.
- Remaining points counter: derived from budget bar — not a separate element.

**Negative Traits panel (Point Buy tab only, below score table):**
- Separator between score table and traits panel.
- Heading: "Negative Traits" at label weight (14px semibold), with subtitle "(optional — each grants bonus points)".
- Two subsections:

  *Preset Mechanical Flaws:*
  - Rendered as a checkbox list. Each row: `[Checkbox] [Name — bold] [penalty description — muted text-sm] [+N pts badge — text-xs rounded bg-secondary px-1.5 py-0.5]`.
  - 10 presets (from RESEARCH.md D-04 list). Max selectable: no limit (any combination).
  - Checkbox component: existing `shadcn/ui checkbox` (`className="accent-amber-500"`).

  *Free-Form Narrative Flaws:*
  - Label: "Free-Form Flaws (up to 2, +2 pts each)".
  - Two `Textarea` fields (`rows={2}`, `placeholder="Describe a character flaw…"`).
  - Each field shows a subtle `+2 pts` badge to its right when the field contains text (≥1 non-space character). Badge: same style as preset badge above.
  - Second field is shown always (not conditionally hidden).

**4d6 Roll tab (extended from existing):**
- Keep existing roll-all button, standard array sidebar, and assignment dropdowns.
- Add per-stat reroll: after roll-all fires, each stat row gains a small `🎲` icon button (`h-6 w-6 p-0`, `variant="ghost"`) to the right of the modifier.
- Reroll button behavior: clicks re-roll 4d6 drop-lowest for that stat; auto-keeps the higher result; the stat value field transitions to `bg-green-950/40 text-green-400` for 600ms via a CSS transition (`transition-colors duration-600`), then returns to default. After one click, the button is replaced by a muted `"✓ used"` text label (`text-xs text-muted-foreground`). No further reroll allowed for that stat.
- The "Roll Stats (4d6)" button label: keep as-is.

**Manual tab (existing content):**
- Keep the existing per-stat override input fields. No change to structure. Tab label "Manual" replaces the existing "Override" link.

## Inventory Tab Encumbrance Display

When `campaign.encumbranceEnabled = true`:
- Top of Inventory tab: banner row `flex justify-between items-center py-2 px-0 border-b border-border mb-3`.
- Left: "Carrying Capacity" label (14px semibold).
- Right: `"X / Y lbs"` in `font-mono text-sm`. X = sum of all equipped/carried item weights (rounded to 1 decimal). Y = 15× STR score (standard 5e encumbrance limit for movement penalty — Phase 7 uses 5× for Encumbered, 10× for Heavily Encumbered per D-27; display shows the absolute limit as Y = 15× STR for the total weight tracker).

  *Status badge (shown only when thresholds crossed):*
  - No badge when X < 5× STR.
  - "Encumbered" badge (amber, `ConditionBadge`-style) when X ≥ 5× STR.
  - "Heavily Encumbered" badge (red, `ConditionBadge`-style) when X ≥ 10× STR.

## Interaction Contracts

### Method Selector Tab Switch (StepAbilityScores)
- User changes method → inline confirmation if any score is assigned → on confirm: reset all six scores to null → render new method's content area.
- No animation on tab switch other than the Tailwind `transition-colors` already applied to tab triggers.

### Point Buy Budget Counter
- Reactive to both score changes (+/− stepper) and flaw toggles/text input.
- Counter value updates immediately (no debounce) as state changes.
- When `spent > budget`: counter turns `text-destructive`; "Next" button disabled; error copy shown below counter.
- When `spent === budget`: counter turns `text-accent-gold`.
- When `spent < budget`: counter is `text-muted-foreground`.

### Per-Stat Reroll (4d6 tab)
- First click: compute new 4d6 drop-lowest; compare to existing value; set stat to max(old, new); apply 600ms green highlight (`bg-green-950/40 text-green-400`); replace 🎲 button with "✓ used" span.
- "Roll Stats (4d6)" re-roll all: does NOT reset per-stat reroll usage (once used = stays used for the session).

### Character Switcher (party mode)
- Clicking a chip: sets `campaignViewStore.activeCharacterId` → all downstream character queries re-run with the new ID → sheet re-renders. No animation beyond Tailwind `transition-colors`.
- Active chip: immediately highlighted (no loading state visible on chip itself; sheet area below shows loading state if query is pending).
- Tab-navigable: chips implement `role="tab"` with `aria-selected`, contained in `role="tablist"`.

### Feat Picker
- Search is local state filter on the in-memory feat list (no IPC call per keystroke).
- Keyboard: arrow keys navigate list; Enter selects; Escape clears selection (if no feat confirmed yet).
- Create custom feat: inline form appears below `Separator` in the list. On "Save Custom Feat": tRPC mutation (`feats.createCustomFeat`); on success, the custom feat appears at the top of the list pre-selected.

### LevelUpModal Progression Guard
- Confirm button disabled until all required sections are complete: HP method chosen AND (if subclass level: subclass selected) AND (if ASI level + feat chosen: feat selected) AND (if level 21+: Epic Boon selected).
- The `confirmDisabled` derived boolean is extended from the existing pattern.

### SRD Library Navigation
- "Rules" button in campaign list header: `react-router-dom` `navigate('/library')`.
- "📖" icon in campaign header: same navigation.
- Back button: `navigate(-1)` (browser history back).
- Section pills: client-side tab switch; no navigation (all content loaded at mount from JSON files).

### World Setup AI Generation
- On campaign creation submit when "AI Generates" is selected: submit button → spinner + "Generating your world…" message → `campaigns.generateWorldBrief` tRPC call → on success: navigate to campaign. The modal stays open and spinner-locked during generation.
- On failure: modal closes anyway (campaign created with null worldBrief); error chip shown via `MutationChipStack` pattern.

- All wizard step transitions: `aria-current="step"` on active `WizardProgress` circle (existing pattern — extended to 4 steps).

- Library section pills: `role="tablist"` / `role="tab"` / `aria-selected`.
- Party size radio cards: `RadioGroup` with `RadioGroupItem` (shadcn `radio-group` component); each card has a visible label.
