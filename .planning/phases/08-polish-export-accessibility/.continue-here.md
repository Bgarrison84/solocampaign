## Critical Anti-Patterns

| Pattern | Description | Severity | Prevention Mechanism |
|---------|-------------|----------|---------------------|
| Edit-after-apply migration drift | party_size and 5 other columns were added to migration 0007 AFTER it had already been applied to the dev DB. Drizzle's migrate() uses hash-based tracking — it saw 0007 as applied and skipped the new ALTER TABLE statements silently. | blocking | Never edit an existing migration file. Always create a new numbered migration. For already-drifted DBs, use repairMissingColumns() in migrate.ts (idempotent PRAGMA table_info check). |
| Silent tRPC errors | Before this session, any tRPC procedure throw was completely invisible in main.log — campaigns.list had been failing for an entire prior session with no log evidence. | advisory | tRPC errorFormatter now added to _base.ts. All procedure errors appear as [tRPC] path: message in main.log. |
| createContext throw → renderer hangs | electron-trpc does NOT wrap createContext in try/catch. Throwing from createContext causes an unhandled promise rejection — the renderer never receives a response and hangs forever in isLoading state (never reaches isError). | advisory | createContext now handles null senderFrame gracefully (allow + return {}) and logs invalid URLs as warn instead of silent throw. |
