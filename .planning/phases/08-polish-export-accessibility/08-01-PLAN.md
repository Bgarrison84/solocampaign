<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| renderer → main (IPC: appPrefs:getInitial) | Renderer requests prefs; main returns plain object. Read-only, no input. |
| renderer → main (tRPC: setFontSize/setHighContrast) | Renderer sends enum/boolean; Zod validates at procedure boundary. |
| appPrefs.dataFolder → initDatabase path | A stored folder path is used to construct the DB file path at startup. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-01 | Tampering | setFontSize/setHighContrast IPC input | mitigate | Zod `z.enum(['small','normal','large'])` and `z.boolean()` reject any other value at the tRPC procedure boundary |
| T-08-02 | Information Disclosure | appPrefs:getInitial returns store | accept | Store holds only fontSize/highContrast/dataFolder — no secrets, no API keys (keys live in secretStorageService, never in appPrefs) |
| T-08-03 | Elevation/Tampering | dataFolder path → DB file path | mitigate | dataFolder is set only via OS folder picker in plan 08-06 (not free-text); this plan only reads it as a path prefix to path.join — no path traversal vector since the value originates from a trusted OS dialog |
| T-08-SC | Tampering | npm install (no new packages this plan) | accept | No package installs in this plan; appPrefs uses already-installed electron-store v10 (audit OK in RESEARCH.md) |
</threat_model>
