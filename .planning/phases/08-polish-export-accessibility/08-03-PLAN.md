<interfaces>
exportImport.ts public API (08-04 extends this file; CampaignCard/CampaignListScreen consume via tRPC):
- exportCampaign(campaignId: string): CampaignExportPayload  // { version:1, type:'campaignExport', exportedAt, data:{...15 tables} }
- importCampaign(payload: CampaignExportPayload): string  // returns new campaignId
- importCampaignOrTemplate(parsed: unknown): { kind: 'campaign'; campaignId: string } | { kind: 'template'; template: StarterTemplatePayload }

15 tables to export (D-01, FK insertion order — RESEARCH.md Section 2 + Landmine 5):
campaigns → custom_feats → sessions → characters → character_resources → character_items → character_spells → character_feats → messages → combatants → campaign_events → quests → npcs → factions → campaign_reference_docs

Raw sqlite access: `const sqlite = getDb().$client` (better-sqlite3 instance). Synchronous prepare().all() / .get(). Transaction via `sqlite.transaction(fn)`.
UUID: `crypto.randomUUID()` (Node built-in). API keys are NOT in SQLite (D-02) — no exclusion logic needed. portraitPath/coverImagePath set to null on import (binary not portable).
</interfaces>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user file → main process (import) | Untrusted JSON read from a user-picked file, parsed and written to the DB |
| renderer → main (export/importJson tRPC) | campaignId (UUID) and dialog triggers cross from renderer |
| main → filesystem (save dialog path) | A dialog-returned path is written to with the export JSON |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-06 | Tampering | imported JSON → DB writes | mitigate | Validate `version === 1` and `type` discriminant BEFORE any DB write; parse inside try/catch; single transaction rolls back on any constraint failure (no partial state) |
| T-08-07 | Information Disclosure | API keys in export | mitigate | API keys live only in secretStorageService, never in SQLite; campaigns table has no api_key column (D-02) — export cannot leak keys |
| T-08-08 | DoS | oversized/malicious import file | mitigate | Reject files > 50MB before readFile; JSON.parse guarded by try/catch with user-facing error |
| T-08-09 | Tampering | export/import file path | mitigate | Paths originate from OS dialog (showSaveDialog/showOpenDialog), not renderer-supplied strings — no path-traversal vector; campaignId validated as UUID by campaignIdSchema |
| T-08-10 | Spoofing | malicious "campaign" file injecting rows | mitigate | All ids regenerated via crypto.randomUUID and FK-remapped — imported data cannot collide with or overwrite existing campaigns |
| T-08-SC | Tampering | `@radix-ui/react-dropdown-menu` install | mitigate | RESEARCH.md Package Legitimacy Audit marks it [ASSUMED] following the same pattern as 10 installed Radix packages; verified legitimate via radix-ui/primitives repo. Installed via shadcn official registry only |
</threat_model>
