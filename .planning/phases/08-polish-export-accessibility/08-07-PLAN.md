    Remove the `aria-live="polite"` (and any `aria-atomic`) from the visible scroll-area div — it currently floods the screen reader on every history load and token.

    Add an ALWAYS-RENDERED off-screen live region as a sibling/first child of the panel root: `<div ref={liveRegionRef} aria-live="polite" aria-atomic="false" aria-label="Story narration" className="sr-only" />` (the .sr-only utility from 08-02; if any doubt about its presence, the inline clip styles from UI-SPEC are an acceptable fallback). It must NOT be conditionally rendered (Landmine 4) and must start empty.

    Add `const liveRegionRef = useRef<HTMLDivElement>(null)` and `const paragraphBufferRef = useRef('')` and a `lastAnnouncedLenRef`/delta tracker so you only feed NEW tokens into the buffer (streamingContent is cumulative — compute the delta since the last effect run). Add a `useEffect` keyed on `[streamingContent, isStreaming]`: append the new delta to `paragraphBufferRef.current`; call `findParagraphBoundary(text)`; if a boundary is found, slice the announcement off the buffer, set `liveRegionRef.current.textContent = ''` then in `requestAnimationFrame` set it to the announcement (double-update so screen readers detect the change). When `isStreaming` transitions to false, flush any remaining trimmed buffer as a final announcement and reset the buffer + delta tracker for the next stream.

    Implement `function findParagraphBoundary(text: string): number`: `const dn = text.indexOf('\n\n'); if (dn > 20) return dn + 2; const se = text.search(/[.!?][\s\n]/); if (se > 60) return se + 2; return -1`.
