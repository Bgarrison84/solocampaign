All sizes in `rem` — they respond to the `--font-scale` CSS custom property set on `<html>`.

| Body | 14px (0.875rem) | 400 (regular) | 1.5 | Settings field labels, data folder path display, menu items, error messages |
| Label | 12px (0.75rem) | 500 (medium) | 1.4 | Section subheadings ("Appearance", "Data"), restart banner text, template pre-fill banner |

- Also set via JS in `main.tsx` before `ReactDOM.createRoot()` using `appPrefs:getInitial` IPC call

### High Contrast Override Theme (`.high-contrast` class on `<html>`)

These values override the base OKLCH tokens. Target: WCAG AA minimum (4.5:1 for normal text ≤18px, 3:1 for large text ≥18px or bold ≥14px) against `--background`.

Rationale for each override: The existing dark palette has adequate contrast for primary foreground (~12:1) but insufficient contrast for `--muted-foreground` (~4.2:1 estimated, below 4.5:1 threshold). The `.high-contrast` block corrects muted elements and increases border visibility.

```css
/* globals.css — add at end of file, after .dark block */
.high-contrast {
  /* Foreground: boost near-white to pure white (existing was already ~12:1; keep) */
  --foreground: oklch(0.98 0.002 260);          /* #f8f8fc — 15:1+ against background */
  --card-foreground: oklch(0.98 0.002 260);
  --popover-foreground: oklch(0.98 0.002 260);
  --secondary-foreground: oklch(0.98 0.002 260);

  /* Muted foreground: raise from L=0.60 to L=0.80 to achieve ≥4.5:1 */
  --muted-foreground: oklch(0.80 0.008 260);    /* ~7:1 against background — passes AA */

  /* Primary accent: increase chroma for visibility */
  --primary: oklch(0.88 0.14 78);               /* brighter warm gold — ~5.5:1 on dark bg */
  --primary-foreground: oklch(0.10 0.004 260);  /* near-black on bright gold */

  /* Borders: make more visible */
  --border: oklch(0.50 0.015 260);              /* ~4.5:1 against background — passes AA for UI components */
  --input: oklch(0.50 0.015 260);

  /* Destructive: boost red to be clearly visible */
  --destructive: oklch(0.72 0.22 27);           /* brighter red — ~5:1 on dark bg */
  --destructive-foreground: oklch(0.10 0.004 260);

  /* Card/secondary surfaces: slightly darker to maintain contrast with foreground */
  --card: oklch(0.12 0.003 260);
  --secondary: oklch(0.20 0.005 260);
  --muted: oklch(0.16 0.004 260);              /* same as background — muted bg loses distinction, acceptable */

  /* Ring: match new primary */
  --ring: oklch(0.88 0.14 78);

  /* Accent gold: boost for decorative use */
  --accent-gold: oklch(0.90 0.20 75);
}
```

**Verification required:** These OKLCH values are computed from palette analysis. Before shipping, verify each foreground/background pair using webaim.org/resources/contrastchecker. Specifically verify:
1. `--muted-foreground` (L=0.80) against `--background` (L=0.16): target ≥4.5:1
2. `--destructive` (L=0.72) against `--background` (L=0.16): target ≥4.5:1
3. `--primary` (L=0.88) against `--background` (L=0.16): target ≥3:1 (large/bold text)
4. `--border` (L=0.50) against `--background` (L=0.16): target ≥3:1 (UI component contrast, WCAG 1.4.11)

**Dark fantasy palette maintained:** The `.high-contrast` block does not switch to a light theme. It is an enhanced-contrast version of the same dark palette. Both `.dark` (always present on `<html>`) and `.high-contrast` apply simultaneously.

## Interaction Contracts

### Settings Screen — Font Size Segmented Control

- **Pattern:** 3 `Button` components in a row (not shadcn ToggleGroup — use 3 Buttons with `variant="outline"` for inactive, `variant="default"` for active)
- **States:** One button active at a time; active = `variant="default"` (gold background); inactive = `variant="outline"`
- **On change:** Immediately update `document.documentElement.style.fontSize` (no reload); persist via tRPC `appPrefs.setFontSize` mutation
- **aria:** Each button has visible text label ("Small" / "Normal" / "Large") — no extra aria-label needed
- **Height:** 36px (Tailwind `h-9`, default Button size)

### Settings Screen — High Contrast Toggle

- **Pattern:** shadcn `Switch` component with `<Label>` and description text
- **Layout:** Label left, Switch right (full-width row with space-between)
- **On change:** Immediately toggle `.high-contrast` class on `document.documentElement`; persist via tRPC mutation
- **aria:** `aria-label="High contrast mode"` on Switch; `id` + `htmlFor` pairing with Label

### Settings Screen — Data Folder

- **Layout:** Read-only `<code>` or monospace `<p>` showing current path; "Change..." button below or inline-right
- **Font:** Monospace (Tailwind `font-mono`) for path display; body weight 400; muted-foreground color
- **Path truncation:** CSS `text-overflow: ellipsis; overflow: hidden; white-space: nowrap` — truncates at right side for Windows paths
- **Change flow:** Click "Change..." → `dialog.showOpenDialog({ properties: ['openDirectory'] })` in main → if confirmed: `sqlite.backup()` → integrity check → `appPrefs.set('dataFolder')` → show restart banner
- **Restart banner:** shadcn `Alert` with `variant="default"` (not destructive); Info icon; shown only when `pendingDataFolderChange` is truthy in local component state

### CampaignCard — 3-Dot Context Menu

- **Trigger position:** `absolute top-2 right-2` within the card's relative container
- **Trigger button:** `Button variant="ghost" size="icon"` with `MoreHorizontal` lucide icon (16px)
- **Click isolation:** `onClick={(e) => e.stopPropagation()}` on trigger button; DropdownMenuContent does not need stopPropagation (Radix portals outside card)
- **Menu width:** `min-w-[180px]` (Tailwind)
- **Menu items:** "Export Campaign (JSON)" + "Export as Starter Template" → `DropdownMenuSeparator` → "Delete Campaign" (className `text-destructive focus:text-destructive`)
- **Delete flow:** onSelect opens existing confirmation Dialog (not inline; reuse current delete dialog pattern)

### PDF Export Button (CharacterSheetTab)

- **Placement:** In the tab header row, right-aligned; `size="sm"` Button with Download icon (`lucide Download`, 14px) + "Export PDF" text
- **Loading state:** Replace icon with `Loader2` lucide icon with `animate-spin`; disable button; change text to "Generating..."
- **Trigger flow:** Click → `characters.exportPdf(activeCharacterId)` tRPC mutation → main process `renderToBuffer()` → `dialog.showSaveDialog()` → `fs.writeFile()` → toast on error only

### Template Pre-fill Banner (CreateCampaignModal)

- **Position:** Top of modal content, before first form section
- **Component:** shadcn `Alert` with `variant="default"` and a FileText or Star lucide icon
- **Copy:** "Pre-filled from template: {templateName}" (heading) + "Review and edit the settings below before creating your campaign." (description)
- **Dismiss:** Not dismissible (stays visible until modal closes or user creates campaign)
- **Shown when:** `initialTemplate` prop is non-null

### ARIA Live Region (StoryScrollPanel)

- **Placement in DOM:** First child inside the StoryScrollPanel root element (before the scroll area); always rendered
- **Visually hidden:** `className="sr-only"` — position absolute, 1px×1px, clipped, off-screen. If Tailwind v4 does not generate `sr-only`, add explicitly to `globals.css`:
  ```css
  .sr-only {
    position: absolute;
    width: 1px;
    height: 1px;
    padding: 0;
    margin: -1px;
    overflow: hidden;
    clip: rect(0, 0, 0, 0);
    white-space: nowrap;
    border: 0;
  }
  ```
- **Attributes:** `aria-live="polite"` `aria-atomic="false"` `aria-label="Story narration"`
- **Update pattern:** Clear to `""` then set to paragraph text in a `requestAnimationFrame` callback (double-update forces screen reader to detect change)
- **Paragraph boundary heuristic (resolved — Claude's discretion):**
  - Primary trigger: `\n\n` (double newline) with minimum 20 characters before the boundary
  - Fallback trigger: `[.!?][\s\n]` (sentence-end) with minimum 60 characters before the boundary
  - On completion of stream (isStreaming becomes false): flush any remaining buffer content as a final announcement
  - Tokens accumulate in a `useRef` buffer — never in React state (avoids re-renders per token)

**Column proportions:** Left 48% / Right 48% / Gap 4% (achieved via `flex: 1` on both columns with `gap: 20pt`)

**Skill list:** 18 rows, 8pt regular. Proficiency dot: filled circle (•) or empty (◦) at left. 4pt gap between dot and skill name. Modifier (+N or -N) right-aligned.

**Currency display:** 5 values in one row (CP / SP / EP / GP / PP), each in a labeled mini-box. 8pt labels, 10pt values.

## Export / Import File Contract

### Campaign Export JSON Structure

```typescript
{
  version: 1,
  type: "campaignExport",
  exportedAt: string,  // ISO 8601
  data: {
    campaign: CampaignRow,          // excludes: no API key column exists
    sessions: SessionRow[],
    messages: MessageRow[],
    characters: CharacterRow[],     // portraitPath set to null on import
    characterResources: CharacterResourceRow[],
    characterItems: CharacterItemRow[],
    characterSpells: CharacterSpellRow[],
    characterFeats: CharacterFeatRow[],
    customFeats: CustomFeatRow[],
    combatants: CombatantRow[],
    campaignEvents: CampaignEventRow[],
    quests: QuestRow[],
    npcs: NpcRow[],
    factions: FactionRow[],
    campaignReferenceDocs: CampaignReferenceDocRow[]
  }
}
```

**Export file naming:** `{campaignName}-export.json` (lowercase, spaces → hyphens)

### Starter Template JSON Structure

```typescript
{
  version: 1,
  type: "starterTemplate",
  exportedAt: string,  // ISO 8601
  name: string,
  worldSetupMode: string,
  worldBrief: string | null,
  worldDocument: string | null,
  dmPersonality: string,
  strictness: string,
  partySize: number,
  encumbranceEnabled: boolean,
  homebrewContent: string | null
}
```

**Template file naming:** `{campaignName}-template.json` (lowercase, spaces → hyphens)

### Import Validation Rules (in order)

1. Parse JSON — catch SyntaxError → "not a valid SoloCampaign export" error
2. Check `payload.version === 1` — mismatch → "created with a newer version" error
3. Check `payload.type === 'campaignExport' || 'starterTemplate'` — neither → "unrecognized format" error
4. For `campaignExport`: run UUID remapping + DB transaction
5. For `starterTemplate`: return template fields to renderer (no DB write at import time)
